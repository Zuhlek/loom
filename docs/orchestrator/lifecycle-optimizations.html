How to read this catalogue

Every concept on this page is something a Loom skill can ship from inside a Claude Code session: prompt wording, dispatch structure, orchestrator logic, eval-harness changes. Anything that requires touching the Anthropic API request body (cache_control, max_tokens, budget_tokens, model selection, Batch API, OTel parallel capture, multi-provider routing) has been removed — that's upstream work and doesn't belong in this planning doc. Experimental research-stage ideas (memory subsystems, speculative tool execution, mixture-of-agents, learned routers) are also out.

Infrastructure — 5 items · plumbing, caching, telemetry, transport

Concept-agnostic. Would apply to any orchestrator. Cache discipline and observability live here; ship-before-anything-else territory.

1.1

Stable-prefix hierarchy

Sort tool JSON keys, freeze tool order, never interpolate dates or UUIDs into the cached system prompt. The cache lookup is byte-for-byte — a single capitalised letter destroys the entire cached prefix.
GitHub Copilot 93% cache-hit rate is achievable.

3.1

Cache-hit-rate dashboard per subagent

Track cache_read / (cache_read + cache_write + input) per subagent. If a subagent is below 60%, something upstream is mutating. Cannot fix what cannot be seen.
ProjectDiscovery hit -59% cost once they could measure it.
InfraFcitedLow costIndirect (large)
PD · GHTE · SD
3.2

Per-phase / per-subagent token attribution

Tag every API call with phase + subagent + task-ID. Anthropic reports 80% of multi-agent performance variance is explained by token usage alone — without attribution this signal is invisible.
Enables 20–50% follow-on wins by exposing where they live.
InfraFcitedLow costIndirect
AMR · TT · LF
3.3

Cost-distribution healthcheck ratio

Augment's ratio: planning + coordinator ≈ 9.8% of total tokens, workers ≈ 70.6%. If the coordinator exceeds that, it is accumulating context it should not. Set as an alarm.
Catches coordinator bloat before it compounds.

OpenTelemetry GenAI semantic conventions

Standard span attributes (gen_ai.system, gen_ai.usage.input_tokens, …) so Loom traces work with Datadog, Honeycomb, Arize, Langfuse without bespoke adapters.
One-time cost, no vendor lock-in.

Prompt & Documentation — 7 items · wording inside SKILL prompts, schemas, return shapes

The change lives in wording, not in code or architecture. Cheapest to ship, safest to roll back. No new agent, no new pipe, no new tool.

1.7

"System-reminder" tail for dynamic data

Dates, IDs, timestamps go into a non-cached tail block of the user turn — never into the cached system prompt. Exactly the pattern Claude Code uses internally.
Prevents silent cache invalidation by date interpolation.
PromptFcitedLow costMedium
PCM · CCC
2.2

Return-by-reference (cite IDs, not content)

Subagents return {decision:"approve", evidence:["US-014","T-031"]}, not pasted artifact text. The coordinator dereferences only if needed. Natural extension of Loom's typed RETURN schemas — Loom is already half-way there.
Already partly in place; tighten the wording.
PromptFestimatedLow costMedium
Loom-native
2.3

"Be concise" prompt tax

Explicit "no preamble, no filler, ≤N words, data only" in every SKILL prompt. Pairs with the max_tokens cap. Typical 10–30% saving per output.
10–30% per output, cumulative across phases.
PromptFcitedLow costLow-Med
RDS
6.3

Structured handoff schema

Replace blob-summary compaction with a typed schema: {intent, changes_made, decisions, next_steps, open_questions}. Sourcegraph Amp killed compaction entirely for this. Factory's eval shows it beats built-in compaction.
Better continuity at lower token cost than free-text summaries.
PromptDcitedMedium costMedium + continuity
FAC · AMP
7.1

Reflexion before retry, not after

Carry forward a previous_attempt_failure_summary block into attempt N+1, not the raw transcript. The model sees what went wrong without the noise of how.
10–20 percentage points on coding benchmarks vs naive retry.
PromptDcitedLow-Med costHigh success rate
RX · PGR
7.3

Different prompt per retry attempt

Do not retry with identical input. Retry-2 adds reflection; retry-3 escalates model tier (Sonnet → Opus). Same loop shape, different prompts.
Reduces "retry with same input, same failure" cycles.
PromptDestimatedLow costMedium
Reflexion · FrugalGPT

Concepts & Phases — 9 items · how phases, subagents and tools are wired together

Loom-shape changes. Affects the orchestrator and dispatcher, not the model. Includes the discipline items — concepts whose value is in not doing something.

4.2

Deterministic pre-agent shell prefetch

Run gh, git diff, ls, lint before the agent loop starts. The agent reads the result as static context instead of issuing tool calls itself.
8–12 KB context shaved per run, 19–62% overall efficiency.
ConceptDcitedLow costHigh
GHTE
4.3

Deterministic skip-LLM relevance gate

A cheap rule-based filter that skips the LLM entirely when the input cannot possibly need it. GitHub's "Security Guard" skips on PRs that do not touch security-sensitive paths. Loom analogue: rule-based gates on Review tasks.
-43% on the gated workload.
ConceptDcitedLow costHigh
GHTE
5.2

Parallel subagent fan-out

Independent subtasks fan out concurrently. Loom's DAG is already the parallelisation plan — the work is verifying the dispatcher actually concurrent-fires on blocked_by ⊆ done ∧ file_scope disjoint, not serial-fires.
Anthropic's research system cites up to 90% time reduction.
ConceptFcitedLow costHigh latency
AMR
6.1

Progressive disclosure / pass-by-ID

Do not dump full spec.md into a subagent. Pass IDs (US-007, Q-014) plus a read_artifact(id) tool. Loom's stable cross-phase IDs make this trivial.
150K → 2K tokens cited by Anthropic (-98.7%).
ConceptDcitedLow costHigh
ACE
6.5

Active context trimming

Chroma's 18-model study: every frontier model degrades well before its context limit; quality follows a U-curve, the middle drops 30%+. Schedule active trims rather than wait for the limit.
22.7% token saving (scheduled) vs 6% (passive).
ConceptDcitedMedium costMedium + accuracy
CR · ML · ATC
6.6

Code-execution-as-tool (MCP "skill")

Instead of N tool calls each producing model-visible JSON, generate one script that calls N tools and only emits the final slice. Anthropic example: 10K-row spreadsheet collapses to 5 rows in the model's context.
98.7% extreme case cited.
ConceptDcitedMedium costHigh
ACE
7.2

Partial-state preservation across retries

On retry, keep the parts of the kanban / artifacts that succeeded — only re-run the failed step. Full re-planning re-pays the planner cost.
Avoids paying the planner cost N times.
ConceptDestimatedMedium costMedium
ATC
16.1

Don't grow the multi-agent count

Cognition's public argument: fewer agents, better context engineering inside one — handoff is lossy. Loom's 5 phases are right-sized. Resist growing to 7 to enforce one new rule.
Preventive — avoids future handoff-tax bloat.
ConceptDcitedFreePreventive
CGN-MA · JX
16.2

Coordinator never reads worker transcripts

Augment's 53.7% reduction came from preventing a coordinator from accumulating workflow history. The coordinator sees only the typed RETURN values from its workers. Loom already removes Edit / Write from the coordinator; the remaining discipline is to remove Read on worker transcript files too.
53.7% cited token reduction.
ConceptFcitedFreeHigh
ATC
