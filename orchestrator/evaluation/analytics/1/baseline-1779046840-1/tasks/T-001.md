# T-001 — Scaffold app workspace with build/test/start scripts

## Goal

Create the empty-but-runnable workspace inside
`.loom/baseline-1779046840-1/app/` such that subsequent tasks can land
server and client code without revisiting toolchain plumbing. After
this task, `npm install` succeeds, `npm test` runs (and reports zero
tests), and the build/start scripts exist and are wired even though
they have nothing to compile yet.

## Single-layer justification

This is a foundation task that cannot be sliced behaviourally — there
is no user-observable surface yet. It exists to unblock T-002 through
T-011. Every story is listed under `satisfies-stories` because its
absence would block delivery of every story's acceptance criteria;
the user-visible assertion happens in the consuming slices.

## Scope

- `package.json` with scripts:
  - `start`: `npm run build && node --enable-source-maps dist/server/index.js`
  - `build`: `tsc -p tsconfig.server.json && node esbuild.config.mjs`
  - `test`: `vitest run`
  - `dev`: `tsx watch src/server/index.ts` (convenience)
- Pinned deps per `design.md § ADR-006`: express, better-sqlite3,
  typescript, vitest, supertest, esbuild, @types/* as needed.
- `tsconfig.json` (strict, shared base) + `tsconfig.server.json`
  emitting to `dist/server/`.
- `esbuild.config.mjs` bundles `src/client/main.ts` → `dist/client/app.js`
  and copies `index.html`, `styles.css` to `dist/client/`.
- `.gitignore` covers `node_modules/`, `dist/`, `bookmarks.db`.
- `src/shared/types.ts` exports `Bookmark` and `ApiError` per
  `design.md § Server module signatures`.

## Acceptance criteria

1. From `app/`, `npm install` exits 0.
2. From `app/`, `npm test` exits 0 (no tests yet — Vitest reports
   "No test files found" is acceptable as long as the script is wired
   and the test runner exists; if a pass-by-default placeholder test
   is needed for `vitest run` to exit 0, add `test/_placeholder.test.ts`
   with a `it.skip('placeholder', () => {})` and remove it once real
   tests land).
3. From `app/`, `npm run build` exits 0 once the source files in
   subsequent tasks exist; at this task's completion `npm run build`
   may fail because `src/server/index.ts` does not exist yet, but the
   script name must be present and runnable in principle (the tsc and
   esbuild configs are valid).
4. `src/shared/types.ts` exports `Bookmark` and `ApiError` matching
   the `design.md` signatures.
5. Every file lives under `.loom/baseline-1779046840-1/app/` per
   Spec constraint #1.

## Behaviour-level test sketch

This task is a scaffold; its behavioural assertion is delegated to
T-012's smoke gate ("install + build + test succeed end-to-end"). The
unit-level assertion for this task is:

- `npm install` succeeds in a fresh checkout.
- `vitest run` exits 0.
- `tsc -p tsconfig.server.json --noEmit` exits 0 against the shared
  types file (proves the tsconfig is valid).
