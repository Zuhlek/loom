# T-002 — SQLite schema and db module with UNIQUE(url)

## Goal

Implement `src/server/db.ts` per `design.md § Data model` and
`§ Server module signatures`. The module owns the SQLite connection,
runs schema migration on boot (`CREATE TABLE IF NOT EXISTS`), and
exposes typed `list`, `insert`, `remove`, `close` functions plus
`openDb(filePath)` and `DuplicateUrlError`.

## Single-layer justification

DB plumbing is below the user-visible surface. Slicing it together
with a route would force both server seams to land in the same task,
making the slice larger and the API contract task harder to verify in
isolation. Justified as foundation because the UNIQUE-at-storage
guarantee (Spec constraint #6, ADR-002) needs a dedicated unit-level
assertion that the route layer cannot make.

## Scope

- `openDb(filePath)`:
  - Opens `better-sqlite3` with the given path (use `:memory:` in tests).
  - Runs `CREATE TABLE IF NOT EXISTS bookmarks (…)` per `design.md`.
  - Returns a `DbHandle` with prepared statements bound to the connection.
- `DbHandle.list()`: `SELECT … FROM bookmarks ORDER BY id DESC`,
  mapping `created_at` → `createdAt` on the way out.
- `DbHandle.insert({ title, url })`: `INSERT … RETURNING …`. Catches
  the `SQLITE_CONSTRAINT_UNIQUE` error code and throws
  `DuplicateUrlError`. Returns the inserted `Bookmark`.
- `DbHandle.remove(id)`: `DELETE FROM bookmarks WHERE id = ?`. Returns
  `true` if `changes > 0`, else `false`.
- `DbHandle.close()`: closes the underlying connection.
- `DuplicateUrlError extends Error` exported by name.

## Acceptance criteria

1. `openDb(':memory:')` returns a working handle with the schema
   present.
2. `insert` returns a `Bookmark` with a numeric `id`, the inserted
   title, the inserted url, and an ISO-8601 `createdAt`.
3. A second `insert` with the same `url` throws `DuplicateUrlError`
   (Spec constraint #6, US-001 AC3 storage-layer enforcement).
4. `remove` returns `true` for an existing id and `false` for a
   missing id.
5. `list` returns rows in `id DESC` order (newest-first) — verified
   by inserting two rows and asserting the order.

## Behaviour-level test sketch

`test/db.test.ts`:

- `insert + list returns the row newest-first` — Insert "a", insert
  "b", assert `list()` returns `[b, a]`. (Supports US-002 AC1 at the
  storage layer.)
- `duplicate url throws DuplicateUrlError` — Insert URL `x`, attempt
  to insert same URL again; expect `DuplicateUrlError`. (Supports
  US-001 AC3 and Spec constraint #6.)
- `remove returns true on hit, false on miss` — Insert row id=1,
  `remove(1)` → `true`; `remove(999)` → `false`. (Supports US-004 AC1
  and AC3.)
- `list on empty db returns []` — Open fresh `:memory:` DB and assert
  `list()` is the empty array. (Supports US-002 AC3 at the storage
  layer.)
