# T-003 — GET /api/bookmarks returns newest-first list

## Goal

Bring up the Express app factory and the read side of the REST
surface: `GET /api/bookmarks` returns every saved bookmark ordered
newest-first. This is the first vertically-shaped server slice; it
introduces `createApp(db, staticRoot)` per ADR-005 so subsequent
route tasks can extend the same factory.

## Scope

- `src/server/index.ts` — export `createApp(db, staticRoot)` that
  builds an Express instance, mounts JSON body parsing, mounts
  `/api/*` routes from `routes.ts`. No `listen()` here; a `main()`
  guarded by `require.main === module` (post-compile) calls
  `openDb()`, `createApp()`, then `app.listen(3000)`.
- `src/server/routes.ts` — exports a `Router` mounted at
  `/api/bookmarks`. The `GET /` route returns
  `200 { bookmarks: Bookmark[] }` via `db.list()`.
- Use `:memory:` SQLite in tests via supertest hitting `createApp`.

## Acceptance criteria

(EARS → server slice for US-002.)

1. WHEN supertest issues `GET /api/bookmarks` against a fresh
   `createApp(:memory:db, …)`, the system SHALL respond
   `200 { bookmarks: [] }`. (US-002 AC3, storage view.)
2. WHEN two bookmarks have been inserted (via `db.insert` in test
   setup), the system SHALL respond `200 { bookmarks: [b, a] }` —
   newest first. (US-002 AC1.)
3. The response Content-Type is `application/json; charset=utf-8`.

## Behaviour-level test sketch

`test/api.test.ts` (new file; subsequent route tasks append to it):

- `GET /api/bookmarks on empty db returns 200 with empty array` —
  Boot `createApp(openDb(':memory:'), …)`; supertest `.get('/api/bookmarks')`
  → status 200, body `{ bookmarks: [] }`.
- `GET /api/bookmarks returns rows newest-first` — Insert two rows
  through the db handle in test setup, then supertest GET; assert the
  body's `bookmarks` are in `id DESC` order with title and url
  matching the inserts.
- `GET /api/bookmarks sets JSON content type` — Assert
  `Content-Type` header matches `application/json`.
