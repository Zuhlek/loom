# T-004 — POST /api/bookmarks with validation and 409 duplicate

## Goal

Add the write side of US-001 at the server: accept a JSON body of
`{ title, url }`, validate it, persist it through `db.insert`, and
return either `201 { bookmark }`, `400 validation` (with `field`), or
`409 duplicate`.

## Scope

- Extend `src/server/routes.ts` with `POST /` (mounted at
  `/api/bookmarks`).
- Validation per `design.md § Server error mapping`:
  - `title`: non-empty after `trim()`, length ≤ 2048.
  - `url`: `new URL(input)` does not throw AND protocol is `http:` or
    `https:`, length ≤ 2048.
  - On failure: `400 { error: { code: 'validation', message, field }}`.
- On `DuplicateUrlError` from `db.insert`: respond
  `409 { error: { code: 'duplicate', message, field: 'url' } }`.
- On success: `201 { bookmark: Bookmark }`.

## Acceptance criteria

(EARS → US-001 ACs.)

1. WHEN supertest POSTs `{ title: 'X', url: 'https://x.test' }`, the
   system SHALL respond `201` with `body.bookmark.id` numeric,
   `title` and `url` echoed, and `createdAt` an ISO-8601 string.
   (US-001 AC1, AC2 server side.)
2. WHEN supertest POSTs the same `url` twice, the second response
   SHALL be `409` with `body.error.code === 'duplicate'`. (US-001 AC3.)
3. WHEN supertest POSTs with empty `title`, the response SHALL be
   `400` with `body.error.code === 'validation'` and
   `body.error.field === 'title'`. (US-001 AC4.)
4. WHEN supertest POSTs with `url: 'not-a-url'`, the response SHALL
   be `400` with `body.error.field === 'url'`. (US-001 AC4.)
5. WHEN supertest POSTs with `url: 'ftp://x.test'`, the response
   SHALL be `400` (only http/https allowed per the design).
6. AFTER a successful POST, GET `/api/bookmarks` SHALL include the
   inserted row at index 0 (newest-first; supports US-001 AC2).

## Behaviour-level test sketch

`test/api.test.ts` (extends T-003):

- `POST returns 201 with the new bookmark` — supertest POST, assert
  status, body shape, and that subsequent GET shows the row first.
- `POST duplicate url returns 409 duplicate` — POST a URL twice;
  assert second is 409 with the duplicate error code.
- `POST empty title returns 400 validation field=title`.
- `POST malformed url returns 400 validation field=url`.
- `POST non-http(s) url returns 400 validation field=url`.
