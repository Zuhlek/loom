# T-006 — Static file serving and index.html shell

## Goal

Mount static file serving for `dist/client/` and ship the HTML shell
the client will hydrate. After this task, the page loads with the
empty markup needed for T-008's render loop and is served from the
same origin as the API (Spec constraint #3).

## Single-layer justification

Static serving and the HTML shell are infrastructure for every client
story but have no user-visible behaviour on their own. The page is
empty until T-008 wires DOM updates. Splitting them out lets T-007
(client api/dom helpers) and the client behaviour slices land
independently.

## Scope

- `src/server/static.ts` exports a middleware that:
  - Serves `dist/client/` for any GET that didn't match `/api/*`.
  - Falls back to `dist/client/index.html` for `/`.
- `createApp(db, staticRoot)` mounts `/api/*` first, then the static
  middleware (route precedence per ADR-001).
- `src/client/index.html` skeleton:
  - `<form id="save-form">` with title input, url input, submit
    button, two error-message slots, one form-level error slot.
  - `<ul id="bookmark-list">` empty container.
  - `<div id="empty-state" hidden>` empty-state node.
  - `<div id="banner" role="status" hidden>` for non-fatal 404 banner.
  - `<script type="module" src="/app.js">`.
- `src/client/styles.css` minimal layout.
- `esbuild.config.mjs` copies `index.html` and `styles.css` into
  `dist/client/` alongside the bundle.

## Acceptance criteria

1. WHEN the server is started and GET `/` is requested, the response
   SHALL be `200` with the contents of `dist/client/index.html`.
2. WHEN GET `/app.js` is requested, the response SHALL be the built
   bundle.
3. WHEN GET `/api/bookmarks` is requested, the response SHALL still
   route to the API (route precedence preserved).
4. `index.html` SHALL contain stable element IDs (`save-form`,
   `bookmark-list`, `empty-state`, `banner`) that the client code
   will look up.

## Behaviour-level test sketch

`test/api.test.ts` (or a new `test/static.test.ts`):

- `GET / returns the html shell` — supertest GET `/`; assert status
  200 and body contains `id="bookmark-list"` and `id="save-form"`.
- `GET /api/bookmarks still routes to API` — supertest GET; assert
  the JSON shape from T-003, proving `/api/*` takes precedence over
  the static fallthrough.
- `GET /app.js returns the bundle` — after `npm run build`, supertest
  GET `/app.js`; assert status 200 and a non-empty body. (May be
  exercised in T-012's smoke gate if running the build inside a unit
  test is awkward.)
