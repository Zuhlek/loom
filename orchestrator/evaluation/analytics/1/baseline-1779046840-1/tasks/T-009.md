# T-009 — Save form with inline validation, duplicate error, optimistic prepend

## Goal

Wire `#save-form` end to end so the user can save a bookmark: validate
on submit, call `createBookmark`, prepend the returned row to the list
on 201, surface inline errors for 400/409 — without reloading the page.
This is the client slice of US-001.

## Scope

- `src/client/main.ts` (extended):
  - On `#save-form` submit: prevent default; read `title` and `url`;
    run client-side validation (non-empty title, `new URL(input)`,
    `http(s):` protocol); on local failure, call `renderFieldError`
    and abort without calling the API.
  - Otherwise call `createBookmark`; on success, prepend the returned
    bookmark to the rendered list and reset the form.
  - On `ApiClientError`:
    - `code === 'validation'`: render the message under the `field`.
    - `code === 'duplicate'`: render the message under the `url` field
      (US-001 AC3).
- `src/client/dom.ts`: a `prependItem(list, item)` helper to add a row
  at the top without re-rendering the whole list.

## Acceptance criteria

(EARS → US-001 ACs at the client.)

1. WHEN the user submits valid input, the system SHALL call
   `createBookmark` and SHALL prepend the returned row to
   `#bookmark-list` without a page reload. (US-001 AC1, AC2.)
2. WHEN the user submits with empty title, the system SHALL render
   an inline error under the title field and SHALL NOT call
   `createBookmark`. (US-001 AC4.)
3. WHEN the user submits with a malformed URL, the system SHALL
   render an inline error under the url field and SHALL NOT call
   `createBookmark`. (US-001 AC4.)
4. WHEN `createBookmark` rejects with `ApiClientError(code='duplicate')`,
   the system SHALL render the duplicate message under the url field.
   (US-001 AC3.)
5. After any successful save, the form fields and error nodes SHALL
   be cleared.

## Behaviour-level test sketch

`test/client/main-save.test.ts` (Vitest jsdom):

- `valid submit calls api and prepends row` — Boot the page, stub
  `listBookmarks` empty, stub `createBookmark` to resolve a fake
  bookmark; programmatically fill and submit the form; assert
  `#bookmark-list` now has exactly one `<li>` with the returned title
  and url, and the inputs are empty.
- `empty title shows validation error and skips api call` — Submit
  with title empty; assert the error node under title is visible,
  `createBookmark` spy was not called.
- `malformed url shows validation error and skips api call`.
- `duplicate url response renders inline duplicate error` — Stub
  `createBookmark` to reject with `ApiClientError(code='duplicate',
   field='url')`; submit; assert the error node under url shows the
  duplicate message, the list is unchanged.
- `successful save clears form and prior errors` — Submit invalid,
  see error; submit valid; assert errors are cleared.
