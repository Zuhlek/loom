# T-010 — Open bookmark in new tab via title link

## Goal

Guarantee that the rendered title link opens the saved URL in a new
browser tab and leaves the bookmarks page itself untouched. This is
the client slice of US-003.

## Scope

The functional pieces already land in T-007's `renderList`: the title
is an `<a target="_blank" rel="noopener noreferrer" href={url}>`.
This task is the behaviour-level assertion of that contract, plus any
necessary refinements to `dom.ts` (e.g. ensuring the URL goes into
`href`, not `textContent`).

## Acceptance criteria

(EARS → US-003 ACs.)

1. WHEN a row is rendered, the title link SHALL carry
   `target="_blank"`, `rel="noopener noreferrer"`, and `href` equal
   to the bookmark's `url`. (US-003 AC1.)
2. WHEN the title link is activated, the bookmarks page itself SHALL
   remain on the same URL with the same DOM (no navigation, no
   reload). (US-003 AC2.)
3. The URL text in the row SHALL render via `textContent`, not
   `innerHTML` (design.md § Security).

## Behaviour-level test sketch

`test/client/main-open.test.ts` (Vitest jsdom):

- `row title link has target=_blank and rel=noopener noreferrer` —
  Render a single bookmark; query for the `<a>`; assert the three
  attributes and `href`.
- `clicking the title link does not navigate the bookmarks page` —
  Render the page, dispatch a click event on the link; assert
  `document.location.href` is unchanged and `#bookmark-list` still
  has the row. (jsdom does not follow `target="_blank"` by default,
  which is what we want.)
- `url text renders via textContent` — Insert a bookmark with a URL
  string containing `<script>`; assert the rendered URL text has no
  `<script>` element in the DOM and the literal text is present.
