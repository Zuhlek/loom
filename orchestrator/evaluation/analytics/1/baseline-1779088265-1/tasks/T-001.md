# T-001 — Bootstrap workspace, tsconfigs, scripts, shared types

## Why

This is the foundation slice. Every later task assumes `./app/` exists with a working TypeScript toolchain (server tsconfig + client tsconfig), pinned production deps (`express`, `better-sqlite3`), pinned dev deps (`typescript`, `esbuild`, `vitest`, `supertest`, `jsdom`, `@types/*`), and the four `npm` scripts (`build:client`, `build:server`, `start`, `test`). It also lands `src/shared/types.ts` so server and client can share the `Bookmark`, `CreateBookmarkInput`, and `ApiErrorBody` types compile-time-only.

This task is a single-layer task (workspace + shared-types) by design — establishing the toolchain is itself the deliverable; downstream tasks are the ones that put it to use. Justification: there is no observable user behaviour to slice through without first creating the workspace, and putting any further code in this task would block T-002 / T-003 from running in parallel.

## Layer coverage

- `workspace` — `package.json`, both tsconfigs, scripts, `.gitignore` for `data/` and `node_modules/` and `dist/` and `public/bundle.js`.
- `shared-types` — `src/shared/types.ts` with `Bookmark`, `CreateBookmarkInput`, `ApiErrorBody`.

## Acceptance criteria

1. Running `npm install` from `.loom/baseline-1779088265-1/app/` exits `0` and produces a `node_modules/` directory.
2. `npx tsc -p tsconfig.json --noEmit` exits `0` against an empty `src/server` / `src/client` (only `src/shared/types.ts` populated).
3. `npx tsc -p tsconfig.client.json --noEmit` exits `0`.
4. `package.json` declares the four scripts exactly as in Design `§ Constraints § Build`.
5. `.gitignore` excludes `node_modules/`, `dist/`, `data/`, and `public/bundle.js`.
6. `src/shared/types.ts` exports `Bookmark`, `CreateBookmarkInput`, `ApiErrorBody` matching Design `§ Shared TypeScript types` verbatim (camelCase `createdAt`).

## Behaviour-level test sketch

This task ships no runtime behaviour; the acceptance gate is purely toolchain. The `vitest` step is satisfied by `vitest run` finding zero test files (exit `0`). The `tsc --noEmit` runs above are the meaningful gate.

A trivial type-only smoke test (`test/shared-types.test.ts`) imports the three types, constructs a value of each via `satisfies`, and asserts at compile time only — keeps Vitest from reporting "no tests found".

## Notes

- Pin Node `>=20` in `package.json` `engines`.
- Server tsconfig: `module: NodeNext`, `target: ES2022`, `strict: true`, `outDir: dist`, `rootDir: src`, includes `src/server/**` + `src/shared/**`.
- Client tsconfig extends server, switches to `module: ESNext`, `target: ES2020`, `include: ['src/client/**', 'src/shared/**']`. `esbuild` does the emit; tsc here is just for type-checking.
- No `nodemon`, no `tsx`, no `ts-node` — out of scope per Design `§ Alternatives considered`.
