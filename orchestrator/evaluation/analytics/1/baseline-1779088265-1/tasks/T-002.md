# T-002 — Open SQLite and run schema migration

## Why

Every later server-side task needs a `Database` handle with the schema applied. Splitting this out keeps T-003 (Express factory) free of SQL concerns and lets T-004 / T-005 / T-007 import a known-good `openDb` for their integration tests.

Single-layer task (`db` only) by design — the migration is the deliverable and it is the only concern. Justification: combining it with the repo (T-004 / T-005 / T-007 work) would couple schema landing to story slices; splitting it lets the repo tasks each focus on one CRUD verb.

## Layer coverage

- `db` — `openDb({ filename })`, schema migration (`CREATE TABLE IF NOT EXISTS bookmarks ...`, `CREATE INDEX IF NOT EXISTS idx_bookmarks_created_at ...`), pragmas (`journal_mode = WAL` for file path; `foreign_keys = ON`).

## Acceptance criteria

1. `openDb({ filename: ':memory:' })` returns a `Database` instance with the `bookmarks` table present, schema matching Design `§ SQLite schema` verbatim (id INTEGER PK AUTOINCREMENT, url TEXT NOT NULL UNIQUE, title TEXT NOT NULL, created_at INTEGER NOT NULL).
2. The compound index `idx_bookmarks_created_at` on `(created_at DESC, id DESC)` is present after open.
3. Calling `openDb` twice on the same file path does not throw and is idempotent on schema.
4. On a file path inside a non-existent directory, `openDb` throws a clear `Error` (whose `message` references the path); it does NOT silently create directories.
5. The function is synchronous (returns a `Database`, not a `Promise`).
6. `journal_mode = WAL` is set when `filename !== ':memory:'`; skipped (or no-op) for `:memory:`.

## Behaviour-level test sketch

`test/db.test.ts` under Vitest:

- `openDb(':memory:')` then introspect via `db.prepare("SELECT name FROM sqlite_master WHERE type='table'").all()` — assert `bookmarks` present.
- Introspect index list via `PRAGMA index_list('bookmarks')` — assert `idx_bookmarks_created_at` present; `PRAGMA index_info('idx_bookmarks_created_at')` confirms columns and DESC order.
- Second call to `openDb(':memory:')` returns a fresh handle, no error (note: `:memory:` does not share state across handles; that is fine).
- `openDb({ filename: '/definitely/does/not/exist/x.db' })` throws.
- Type-level: `openDb` return type is `Database` from `better-sqlite3`, not `Promise<Database>`.

These directly back US-001 AC1 (the `created_at` integer column) and US-002 AC1 (the index that makes newest-first ordering deterministic).

## Notes

- Migration is inline in `openDb` — no `migrations/` directory, no `umzug`, per Design `§ Migrations`.
- `better-sqlite3` rethrows ENOENT for missing parent directories — surface its `Error` as-is.
- Do not log to stdout/stderr in `openDb` — that is `index.ts`'s job (T-008).
