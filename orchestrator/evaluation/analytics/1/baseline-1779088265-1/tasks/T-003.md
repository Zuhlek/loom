# T-003 — Express app factory with JSON middleware and error handler

## Why

Establishes `createApp({ repo, staticDir })` as the single seam between the runtime (T-008) and the test suite (every later integration test). Lands the empty `routes.ts` shell with the JSON body parser, the static-file middleware, the 404 fallthrough, and the error middleware that maps `DuplicateUrlError` → `409` and unknown errors → `500`. Story-slice tasks (T-004 / T-005 / T-007) plug their handlers into this skeleton.

This task does NOT yet implement any of the three bookmark routes — those land in their slice tasks. It DOES land the error middleware so the slice tasks need only throw the right error.

## Layer coverage

- `app` — `createApp` factory; `express.json({ limit: '10kb' })`; `express.static(staticDir)`; mounting `routes.ts`; error middleware.
- `routes` — empty `routes.ts` exporting `createRoutes(repo)` returning an `express.Router`. Handlers are added in T-004/T-005/T-007.

## Acceptance criteria

1. `createApp({ repo, staticDir })` returns an Express app instance. It does NOT call `.listen`.
2. `POST` with a body > 10kb returns `413` (Express's default for the JSON parser limit) — not handled by us, just confirm we use the limit.
3. A request to an unhandled API path (e.g. `GET /api/unknown`) returns `404` with the `{error: {code, message}}` envelope.
4. The error middleware: when a handler throws `DuplicateUrlError`, responds `409` with `{error: {code: 'DUPLICATE_URL', message}}`. When a handler throws anything else, responds `500` `{error: {code: 'INTERNAL', message: 'Internal error'}}` and writes the original error to `stderr`.
5. `express.static(staticDir)` serves files from the passed directory at `/` and `/public/*` (per Design `§ HTTP API`). A request for a missing static file returns `404`.
6. The factory wires `createRoutes(repo)` under `/api` so later tasks add `GET /bookmarks`, `POST /bookmarks`, `DELETE /bookmarks/:id` to that router.

## Behaviour-level test sketch

`test/app-factory.test.ts` under Vitest + `supertest`:

- Pass a stub `repo` whose methods throw if called (T-003 should not call them).
- `GET /api/unknown` → `404`, body matches error envelope.
- A test-only route that throws `new DuplicateUrlError('dup')` → `409` with code `DUPLICATE_URL`.
- A test-only route that throws `new Error('boom')` → `500` with code `INTERNAL`; `console.error` was called once with the original error.
- A test-only route registered after `createApp` that calls `next()` with no error falls through to the 404 handler.
- Static: write a temp file `staticDir/probe.txt`, `GET /probe.txt` → `200` with the contents.

These directly back the HTTP envelope under which US-001..US-004 are delivered.

## Notes

- `DuplicateUrlError` lives in `bookmarks-repo.ts` (Design `§ Repository contract`) — T-003 imports the class only for the `instanceof` check in the error middleware. The repo file may need a minimal stub landed in T-003 (just the class) so the import resolves; the real repo lands in T-004.
- The error middleware is the FINAL middleware — registered after `createRoutes(repo)`. The 404 handler is a non-error middleware registered just before it.
- Do not introduce CORS, helmet, morgan, or any other middleware — Design `§ Constraints § Libraries (production)` lists only `express` and `better-sqlite3`.
