# T-008 — Boot process, static shell, smoke gate

## Why

Closes the loop on the `npm start` / `npm test` contract from the seed. Lands the process entrypoint (`server/index.ts`) that wires `openDb` → repo → `createApp` → `app.listen(3000)`, the static `index.html` shell that loads `public/bundle.js`, and the client boot file (`client/main.ts`) that mounts the form and loads the list. Finishes with the smoke gate in `tests.md` — the end-to-end check that the four stories work together from a clean install.

## Layer coverage

- `boot` — `server/index.ts`: resolves `data/bookmarks.db` path next to the server, calls `openDb`, builds `repo`, calls `createApp({ repo, staticDir: <abs path to ./public> })`, calls `.listen(3000)`. On `openDb` throw, log path to stderr and `process.exit(1)`.
- `static` — `public/index.html` with form (URL input, title input, submit button, error region) and list container; `<script type="module" src="/bundle.js"></script>`. `public/styles.css` with `prefers-color-scheme` support so dark mode falls out for free.
- `client-boot` (lives under `boot` layer for accounting) — `src/client/main.ts`: on `DOMContentLoaded`, wire `form.ts` onto the form element, call `loadAndRender(listRoot)` once, pass the saved/deleted callbacks.
- `smoke` — `test/smoke.test.ts` (Vitest test that spawns `node dist/server/index.js` against a temp data dir, hits the API, asserts behaviour) AND the manual `npm start` + `curl` script captured in `tests.md` `§ Smoke gate (explicit)`.

## Acceptance criteria

1. `npm install && npm test` from `.loom/baseline-1779088265-1/app/` exits `0` with every Vitest file passing.
2. `npm start` boots a server on `:3000` within 5 seconds. `GET /` returns the HTML shell whose body includes the form and list container. `GET /bundle.js` returns the esbuild output.
3. `POST /api/bookmarks {url, title}` → `201`; `GET /api/bookmarks` includes it. A second `npm start` (after killing the first) sees the same row — persistence across restarts (Spec invariant).
4. Killing the server cleanly leaves `data/bookmarks.db` (and `bookmarks.db-wal`, `bookmarks.db-shm` from WAL mode) under `./app/data/`. Nothing is written outside `./app/`.
5. On a path where SQLite cannot open the file (e.g. `data/` removed and `mkdir` fails), the process exits `1` with a stderr message naming the path.
6. The `prefers-color-scheme: dark` media query is honoured by `styles.css` — Spec explicitly allowed dark mode "if it falls out of CSS for free".
7. All eight tasks' acceptance criteria pass end-to-end against the booted server.

## Behaviour-level test sketch

`test/smoke.test.ts` (Vitest, slower than unit tests; runs as part of `npm test` but tagged so it can be skipped via env if needed):

- Spawn `node dist/server/index.js` with `DATA_DIR=<tmp>` env var; wait for `:3000` (poll TCP).
- Hit each endpoint: `GET /api/bookmarks` (empty), `POST /api/bookmarks` (201), `GET /api/bookmarks` (one row), `DELETE` (204), `GET /api/bookmarks` (empty).
- Kill the process; respawn against the same `DATA_DIR`; insert a row; kill; respawn; assert the row is still there. This covers Spec `§ Runtime invariants: SQLite file SHALL survive process restarts`.
- File-system assertion: after the test, no path outside `<tmp>` and `./app/dist/` is newer than the test start. Workspace-isolation invariant.

The cross-cutting `no-innerHTML` test (from `tests.md`) also runs here — it greps `src/client/**` and asserts zero matches.

## Notes

- `index.ts` reads the DB path from `process.env.DATA_DIR ?? path.resolve(__dirname, '../../data')` so the smoke test can point it at a tmp dir without monkey-patching.
- `app.listen(3000)` callback logs `Bookmarks listening on http://localhost:3000` once; nothing else logged in steady state.
- `index.html` is hand-written, ~30 lines: `<!doctype html>`, `<meta charset>`, `<meta viewport>`, `<title>Bookmarks</title>`, `<link rel="stylesheet" href="/styles.css">`, the form, the list container, `<script type="module" src="/bundle.js">`.
- `styles.css`: one `:root` with light-mode CSS variables, one `@media (prefers-color-scheme: dark)` overriding them. No toggle.
- Server `build:client` runs before `build:server` in the `start` script so `public/bundle.js` exists when the server starts.
