# T-001 — Workspace scaffold and run contracts

## Goal

Stand up the `app/` workspace under `.loom/baseline-1779088275-1/app/` so that `npm install`, `npm start`, and `npm test` are runnable from `./app/` per the spec's run contract. This is the only intentionally single-layer task in the plan; it is justified below.

## Single-layer justification

A vertically-sliced "scaffold" task is impossible: until the workspace exists, no behavior can be wired through it. Every behavior-bearing task (`T-002`..`T-005`) depends on this scaffold to even hold its source files. The scaffold itself owns no acceptance criterion in `spec.md` — it owns the **constraint envelope** (`Run contract`, `Workspace isolation`, `Stack lock`) that the spec demands as universal preconditions. Treating it as a foundation rather than smearing it across the four behavior tasks keeps each behavior task's diff legible.

## Scope

- Initialize `app/package.json` with `start`, `prestart` (esbuild bundle), and `test` (`tsc --noEmit && vitest run`) scripts.
- Pin dependencies: `express`, `better-sqlite3`. Dev deps: `typescript`, `tsx`, `esbuild`, `vitest`, `supertest`, `happy-dom`, `@types/express`, `@types/better-sqlite3`, `@types/supertest`, `@types/node`.
- Initialize `app/tsconfig.json` for ES2022 target, `module: "ES2022"`, `moduleResolution: "Bundler"`, `strict: true`, `noEmit: true`, with `src/` and `tests/` in `include`.
- Initialize `app/vitest.config.ts` with the default Node environment and globals on; happy-dom is opted-in per-file via the `// @vitest-environment` pragma.
- Create the minimum source tree the later tasks will extend:
  - `app/public/index.html` — bare HTML doc with the `#add-form` and `#bookmarks` mount nodes wired to `/dist/main.js` and `/style.css`. Empty mount nodes only — list and form behavior arrive in T-002/T-003.
  - `app/public/style.css` — minimal layout with a `@media (prefers-color-scheme: dark)` block. No toggle.
  - `app/src/server/index.ts` — boots Express on `process.env.PORT ?? 3000`, mounts `express.static('public')`, mounts `express.static('dist')`, mounts a placeholder `/api` router that 501s every method until T-002 replaces it. Installs `SIGINT` / `SIGTERM` graceful shutdown.
  - `app/src/client/main.ts` — empty `DOMContentLoaded` handler. Real wiring arrives in T-002.
- Create `app/.gitignore` listing `node_modules/`, `dist/`, `data/`, `*.db`, `*.db-journal`.
- Verify the smoke contract: `npm install`, `npm start` (in a one-shot child process from a test) returns HTTP 200 on `GET /` serving `index.html`. This proves the workspace-isolation directive and the run contract before any behavior lands on top.

## Out of scope

- API endpoints beyond the 501 placeholder router.
- DB schema, repo, validation. They arrive when a story needs them.
- Real frontend behavior. The HTML mount nodes exist but the client `main.ts` is a no-op.

## Behavior-level test sketch

No user story is satisfied by T-001, so there is no EARS clause to derive a test from. The acceptance gates are constraint-level:

- `npm install` exits zero from `app/`.
- `tsc --noEmit` exits zero.
- `vitest run` exits zero (with zero specs, or with a placeholder spec that asserts `1 + 1 === 2` — kept until T-002 lands real specs).
- The smoke spec (`tests/smoke/run.test.ts`, guarded by `SMOKE=1`) spawns `npm start`, polls `GET http://localhost:3000/` until it returns 200 with HTML containing `id="bookmarks"`, then kills the child cleanly.
- The `data/` directory and `bookmarks.db` MUST NOT appear under the repo root or sibling workspaces after `npm start` — Build verifies workspace isolation by listing the parent directory after the smoke spec ends.

## Notes for Build

- The spec's `npm start` contract requires the server to listen on port 3000 by default. Keep that the default; allow `PORT` env override for the smoke test to use an ephemeral port if necessary.
- The 501-placeholder router is replaced wholesale in T-002 — don't carry it forward.
