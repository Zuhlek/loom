# T-003 — Save a bookmark end-to-end with inline validation and dedupe

## Goal

Deliver US-001 end-to-end: the user types a title and URL into the add form, submits, and either sees the new bookmark appear in the list (success) or sees an inline error (invalid title, invalid URL, or duplicate URL). The full vertical slice: validation → repo insert → 201 / 400 / 409 routes → API client → render error → main wiring → refetch → list update.

## Scope

### Server

- `src/server/validation.ts`: `parseTitle(raw)` trims whitespace, rejects non-string and empty-after-trim with `code: "INVALID_TITLE", field: "title"`. `parseUrl(raw)` rejects non-string; constructs `new URL(raw)`; rejects unless the protocol is `http:` or `https:`; returns the un-normalized original string on success. Failure shape: `code: "INVALID_URL", field: "url"`.
- `src/server/repo.ts`: `insert({ title, url })` runs `INSERT INTO bookmarks (title, url, created_at) VALUES (?, ?, ?)` with `new Date().toISOString()` as the timestamp; catches the SQLite `SQLITE_CONSTRAINT_UNIQUE` error and throws `DuplicateUrlError`. (Schema-level `UNIQUE(url)` was already created in T-002.)
- `src/server/routes.ts`: `POST /api/bookmarks`. Reads JSON body. Runs `parseTitle` and `parseUrl`; on `ok: false` returns 400 with the `code` and `field` from the parse result. On valid input calls `repo.insert`; returns 201 `{ bookmark }`. Catches `DuplicateUrlError` and returns 409 `{ error: { code: "DUPLICATE_URL", field: "url", message: ... } }`. Bubbles other errors to the single 500 error middleware.

### Frontend

- `src/client/api.ts`: `createBookmark(input): Promise<Bookmark>`. On non-2xx, throws `ApiError` populated from the response JSON's `error` shape.
- `src/client/render.ts`: `showFormError(form, field, message)` and `clearFormErrors(form)`. The error nodes are inline inside the form, positioned under the field that errored (`field: "form"` is used for the dedupe/internal-error case where no single field carries the error — design says dedupe is `field: "url"`, so the "form"-level path is for the 500 case only).
- `src/client/main.ts`: wires the `#add-form` submit handler. On submit: prevent default, disable the submit button, `clearFormErrors`, call `createBookmark`. On success: call `refresh()` (the function landed in T-002) which refetches `GET /api/bookmarks` and re-renders. On `ApiError`: call `showFormError(form, err.field ?? "form", message)`. The error message text is implementation detail per `decisions.md` notes. Re-enable the submit button in `finally`. Wire an `input` event on the form that calls `clearFormErrors` for the field that produced an error on the previous submit (the spec's "inline error text is cleared on the next input event" behavior).

### Tests (Vitest)

- `tests/unit/validation.test.ts`:
  - `parseTitle("")` → `ok: false, code: "INVALID_TITLE"`.
  - `parseTitle("   ")` → `ok: false, code: "INVALID_TITLE"`.
  - `parseTitle("  hello  ")` → `ok: true, value: "hello"`.
  - `parseTitle(42)` → `ok: false`.
  - `parseUrl("not-a-url")` → `ok: false, code: "INVALID_URL"`.
  - `parseUrl("ftp://example.com")` → `ok: false`.
  - `parseUrl("javascript:alert(1)")` → `ok: false`.
  - `parseUrl("http://example.com/")` → `ok: true, value: "http://example.com/"` (un-normalized).
  - `parseUrl("https://example.com")` → `ok: true`.
- `tests/unit/repo.test.ts` (extends T-002's file):
  - `insert({ title: "A", url: "http://x.example/" })` persists a row with `created_at` non-null; a subsequent `listAll()` includes it.
  - A second `insert` with the same URL throws `DuplicateUrlError`; the original row is untouched.
- `tests/unit/render.test.ts` (extends T-002's file, happy-dom):
  - `showFormError(form, "title", "Title is required")` inserts an inline error node next to the title input.
  - `clearFormErrors(form)` removes those nodes.
- `tests/http/bookmarks.test.ts` (extends T-002's file):
  - `POST /api/bookmarks` with `{ title: "A", url: "http://x/" }` returns 201; subsequent `GET` lists it.
  - `POST` with `{ title: "  ", url: "http://x/" }` returns 400 `INVALID_TITLE`.
  - `POST` with `{ title: "A", url: "javascript:1" }` returns 400 `INVALID_URL`.
  - Second `POST` with the same URL returns 409 `DUPLICATE_URL`; only one row is persisted.
- `tests/client/api.test.ts`:
  - With `fetch` mocked to return 409 with the error envelope, `createBookmark` rejects with an `ApiError` whose `status === 409`, `code === "DUPLICATE_URL"`, `field === "url"`.
  - With `fetch` rejecting, `createBookmark` rejects with `ApiError` whose `status === 0`, `code === "NETWORK"`.

## Notes for Build

- Keep `parseUrl`'s success value byte-equal to the input (no normalization). The dedupe contract is "exact string equality on the submitted URL" (`spec.md` `## Constraints` → "Duplicate detection key").
- The `decisions.md` notes flag exact error-message wording as implementation detail. Pick clear, short strings (e.g. "Title is required", "Enter a valid http:// or https:// URL", "This URL is already saved").
