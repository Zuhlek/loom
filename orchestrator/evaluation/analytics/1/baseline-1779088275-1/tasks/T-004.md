# T-004 — Open a bookmark in a new tab from the list

## Goal

Deliver US-003: clicking (or keyboard-activating) the title of a bookmark in the list opens the bookmark's URL in a new browser tab, while the originating tab keeps the list visible.

## Single-task vs. folded-into-T-002 rationale

The list-render path already lands in T-002. The new-tab affordance is a small but distinct surface — `target="_blank"` plus the `rel="noopener noreferrer"` safety attribute — that the spec calls out as its own acceptance criterion and that has its own failure mode (omit `rel` and you leak `window.opener` to the user-supplied URL). Splitting it from T-002 keeps the test diff and the review focus tight.

## Scope

- `src/client/render.ts`: update `renderList` so each list item's title `<a>` element carries:
  - `href` = bookmark URL
  - `target="_blank"`
  - `rel="noopener noreferrer"`
- `src/client/main.ts`: no behavior change. The anchor is plain HTML; the browser handles the new-tab navigation. No JS handler intercepts the click.
- `tests/unit/render.test.ts` (extends T-002's file, happy-dom):
  - For a rendered list of one bookmark, query the title anchor and assert `href === bookmark.url`.
  - Assert `target === "_blank"`.
  - Assert `rel === "noopener noreferrer"`.
  - Assert the anchor is reachable via keyboard focus (default `<a href>` behavior — confirmed by `anchor.tabIndex === 0` or by the absence of `tabindex="-1"`).

## EARS coverage map

| EARS clause | Test |
| --- | --- |
| AC-1: activating the title navigates to the URL in a new browser tab | `render.test.ts` — `target="_blank"` + `href === url` (full real-browser navigation is outside the `node-test` harness; this is the strongest assertion the harness can make) |
| AC-2: originating tab does not navigate away | `render.test.ts` — `target="_blank"` plus the absence of any `click` handler that calls `event.preventDefault` and then navigates. `rel="noopener noreferrer"` is the platform contract that prevents the new tab from steering the originator |

## Notes for Build

- Do not register a `click` listener on the anchor — the browser's native handling of `target="_blank"` is what delivers AC-1 and AC-2. A JS-driven `window.open` is unnecessary and would also need keyboard-activation handling.
- `rel="noopener noreferrer"` is the security baseline; do not relax it. The user-supplied URL is not trusted.
