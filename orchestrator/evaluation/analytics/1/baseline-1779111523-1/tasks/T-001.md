# T-001 — Bootstrap app workspace and serve empty bookmark list

## Observable behaviour

Running `npm start` from `app/` boots a Node + Express process on
`http://localhost:3000`. Opening the URL in a browser shows the
bookmarks page rendered from the same origin, which fetches
`GET /api/bookmarks`, receives an empty list, and displays the
empty-state message. The SQLite file at `app/data/bookmarks.sqlite`
is created on first start with the schema from
`design.md ## Data model`. `npm test` runs from `app/` to a clean
exit code.

## Stories satisfied

- **US-002** — list view renders newest-first; empty state shown when
  list is empty. AC1 (load `/` renders bookmarks newest-first; trivially
  satisfied with zero rows), AC2 (each row shows title and URL; tested
  via render-helper unit), AC3 (empty-state message when empty).
- **US-005** (structural part) — the on-disk SQLite file at
  `app/data/bookmarks.sqlite` is the canonical store (AC2). The
  acceptance gate for AC1 (restart-survives) is T-005.

## Scope

In:

- Workspace scaffolding under `.loom/baseline-1779111523-1/app/`:
  `package.json` with `start` and `test` scripts, `tsconfig.json`,
  `.gitignore` (covers `node_modules`, `app/data/*.sqlite`,
  `public/app.js`).
- `esbuild` glue (`scripts/build-client.mjs` or inline in package
  scripts) compiling `src/client/main.ts` → `public/app.js`. `npm
  start` ensures the bundle exists (rebuild if stale) before binding.
- Server entry `src/server/index.ts`: exports `createApp(repo)` and
  `start(port?)`, mounts static at `/` (and `/public/*`) and JSON at
  `/api/bookmarks`, opens DB via `db/connection.ts`.
- DB bootstrap `src/server/db/connection.ts`: resolves path from
  `BOOKMARKS_DB_PATH` env var or `app/data/bookmarks.sqlite` default;
  runs the `CREATE TABLE IF NOT EXISTS bookmarks (...)` migration on
  open per `design.md ## Data model`.
- Repository `src/server/db/bookmarks.ts`: implements `list()`
  returning `Bookmark[]` newest-first by `(created_at DESC, id DESC)`.
  `create()` and `delete()` get stubs that throw `Error('not yet
  implemented')` — wired in T-002 / T-004.
- HTTP route `GET /api/bookmarks`: returns `200 { bookmarks: [] }`
  shape; calls `repo.list()`.
- Static UI: `public/index.html` skeleton with a list container,
  `<script src="/public/app.js" defer></script>`, baseline CSS.
- Client `src/client/main.ts`: on `DOMContentLoaded`, fetches
  `/api/bookmarks`, calls `render(state)`. When `bookmarks.length === 0`,
  shows an empty-state message ("No bookmarks yet."). Exposes
  `render()` and an HTML-escape helper as testable exports.

Out:

- Save form (T-002), Open semantics (T-003), Delete control (T-004),
  restart-persistence test (T-005).

## Test sketch (Vitest)

`tests/repository.list.test.ts`:

- Given a freshly opened temp SQLite DB, `repo.list()` returns `[]`.
- After three direct `INSERT`s with monotonically increasing
  `created_at`, `repo.list()` returns them newest-first (last inserted
  first).

`tests/routes.list.test.ts`:

- `GET /api/bookmarks` against a freshly built app returns `200` and
  body `{ bookmarks: [] }`.
- After two `INSERT`s, `GET /api/bookmarks` returns both, newest-first,
  each with `id`, `title`, `url`, `createdAt`.
- `GET /` returns `200`, `Content-Type: text/html`, body contains a
  `<script>` tag pointing at the bundled client.

`tests/client.render.test.ts`:

- `render({ bookmarks: [] })` produces the empty-state message string
  / element ("No bookmarks yet." or equivalent literal).
- `render({ bookmarks: [{id:1,title:"T",url:"https://e.x",createdAt:"..."}] })`
  produces markup containing the title and the URL, each escaped.
- HTML-escape helper turns `<script>` into `&lt;script&gt;`.

## Acceptance

- `npm ci && npm test` in `app/` exits 0.
- `npm start` boots, `curl -fsS http://localhost:3000/` returns 200
  HTML, `curl -fsS http://localhost:3000/api/bookmarks` returns
  `{"bookmarks":[]}`.
- All `tests/repository.list.test.ts`, `tests/routes.list.test.ts`,
  and `tests/client.render.test.ts` pass.
