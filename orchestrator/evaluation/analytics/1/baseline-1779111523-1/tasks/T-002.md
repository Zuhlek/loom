# T-002 â€” Save a URL with title via POST and render newest-first

## Observable behaviour

The bookmarks page renders a save form with a URL field, a title
field, and a submit button. Submitting a non-empty URL and title
posts to `POST /api/bookmarks`; the server persists a new row and
returns `201` with the created bookmark; the client re-fetches the
list and re-renders, the new bookmark appearing at the top. Submitting
a URL that already exists yields a `409 duplicate_url` and an inline
error "Already bookmarked." attached to the URL input. Submitting
empty or whitespace-only fields yields `400 invalid_input` with
field-level error.

## Stories satisfied

- **US-001** â€” Save a URL with a title.
  - AC1: `POST` with valid input persists and the new bookmark
    appears at the top.
  - AC2: duplicate URL is rejected with inline error; no duplicate

  - AC3: empty / whitespace URL or title is rejected with inline

## Scope

In:

- `src/server/validation.ts`: `normaliseInput({ title, url })` trims
  inputs, throws `ValidationError('title' | 'url', msg)` on empty /
  whitespace-only or on `new URL(url)` throwing. WHATWG `URL`
  constructor per ADR-006; no scheme allow-listing.
- Repository `create(input)`:
  - Calls `INSERT INTO bookmarks (title, url) VALUES (?, ?)`.
  - On `SQLITE_CONSTRAINT_UNIQUE`, throws `DuplicateUrlError`.
  - Returns the inserted `Bookmark` DTO (id, title, url, createdAt).
- HTTP route `POST /api/bookmarks`:
  - Parses JSON body; calls `normaliseInput()`.
  - On `ValidationError`: `400 { error: { code: 'invalid_input',
    message, field } }`.
  - On `DuplicateUrlError`: `409 { error: { code: 'duplicate_url',
    message: 'Already bookmarked.', field: 'url' } }`.
  - On success: `201 { bookmark: Bookmark }`.
- Client save form:
  - `index.html` gains the form markup (URL input, title input,
    submit button, an error placeholder near each input).
  - `main.ts` adds `saveBookmark()`: serialises form, `fetch('POST
    /api/bookmarks', â€¦)`, on 2xx calls `loadBookmarks()` to re-render;
    on 400/409 attaches `error.message` to the field named by
    `error.field`.
  - Inline error wording for duplicates is exactly "Already
    bookmarked." (per `design.md Â§ Open ambiguity`).

Out:

- Click-to-open semantics on list rows (T-003).
- Delete control (T-004).

## Test sketch (Vitest)

`tests/validation.test.ts`:

- `normaliseInput({ title: ' x ', url: 'https://e.x' })` returns
  `{ title: 'x', url: 'https://e.x' }`.
- Empty title throws `ValidationError` with `field === 'title'`.
- Empty URL throws `ValidationError` with `field === 'url'`.
- `'not a url'` throws `ValidationError` with `field === 'url'`.
- `'  '` (whitespace) for either field throws.

`tests/repository.create.test.ts`:

- `repo.create({ title: 't', url: 'https://e.x' })` returns a
  `Bookmark` with `id`, ISO-8601 `createdAt`, trimmed strings.
- Calling `create` twice with the same URL throws
  `DuplicateUrlError`.
- `repo.list()` after two creates returns them newest-first.

`tests/routes.create.test.ts`:

- `POST /api/bookmarks` with valid body returns 201; body matches
  `{ bookmark: { id, title, url, createdAt } }`.
- `POST` with same URL twice: second returns 409 with body
  `{ error: { code: 'duplicate_url', message: 'Already bookmarked.',
    field: 'url' } }`.
- `POST` with empty url or empty title returns 400 with body
  `{ error: { code: 'invalid_input', message, field } }`.
- `POST` with `url: 'not a url'` returns 400 with `field: 'url'`.
- After a successful POST, `GET /api/bookmarks` returns the new
  bookmark at index 0.
