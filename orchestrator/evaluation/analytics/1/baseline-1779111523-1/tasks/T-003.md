# T-003 — Open bookmark in new tab from list row

## Observable behaviour

Each rendered bookmark row presents the title (and URL) as a
clickable anchor pointing at the saved URL with `target="_blank"`
and `rel="noopener noreferrer"`. Clicking opens the bookmark in a new
browser tab; the original tab keeps showing the bookmarks list — no
in-page navigation, no state loss.

## Stories satisfied

- **US-003** — Open a saved bookmark in a new tab.
  - AC1: clicking the title or URL opens the URL in a new browser
    tab. Asserted by `target="_blank"` on the anchor (browser-honoured
    contract).
  - AC2: the original tab preserves the list. Asserted by the anchor
    not being a `javascript:` handler and not bound to a `preventDefault`
    + `location.assign` workaround — the default browser semantics
    with `target="_blank"` leave the current tab untouched.

## Scope

In:

- Update `render()` in `src/client/main.ts` so each row contains:
  - an `<a>` element with `href` = bookmark URL (escaped),
    `target="_blank"`, `rel="noopener noreferrer"`, wrapping the
    title text; and a second `<a>` with the same attributes wrapping
    the displayed URL (the URL is also click-to-open). Both anchors
    point at the same target.
  - All attribute values are escaped through the existing HTML-escape
    helper.
- `public/styles.css` baseline styling for the anchor (no underline
  jitter; falls back gracefully).
- `index.html` no functional change beyond CSS hookup if needed.

Out:

- The delete button (T-004).
- Persistence test (T-005).

## Test sketch (Vitest)

`tests/client.open.test.ts`:

- `render({ bookmarks: [{ id:1, title:'Example', url:'https://example.com/', createdAt:'...' }] })`
  produces markup containing an `<a>` whose `href` is
  `https://example.com/`, with `target="_blank"` and
  `rel="noopener noreferrer"`.
- The anchor's `target` and `rel` attributes are present on *both*
  the title anchor and the URL anchor.
- HTML-dangerous title or URL is escaped:
  `render({ bookmarks: [{ ..., title:'<x>', url:'https://e.x/?a=<b>' }] })`
  yields markup where `<x>` is rendered as `&lt;x&gt;` in the visible
  text and the URL attribute is properly attribute-escaped.
- The rendered anchor markup does NOT contain `onclick`,
  `javascript:`, or any inline JS handler.

## Acceptance

- New `tests/client.open.test.ts` passes.
- Existing T-001 / T-002 tests still pass.
- Manual `npm start` smoke: clicking a bookmark opens a new browser
  tab; original tab stays on the list view.

## Notes / constraints

- `rel="noopener noreferrer"` is mandatory whenever `target="_blank"`
  is used: removes the legacy `window.opener` leak path. This is the
  only non-trivial decision in T-003 and is captured here so Review
  has a concrete assertion to check.
