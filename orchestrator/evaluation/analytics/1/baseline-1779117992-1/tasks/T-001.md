# T-001 — Scaffold ./app/ package, tsconfig, esbuild, Vitest config

## Goal

Lay down the minimum project metadata inside `./app/` so every later task
has a runnable `npm test` and `npm start` skeleton. No source files; no
behaviour.

## Scope

In:
- `package.json` with scripts `start` (build then `node dist/server.js`),
  `build` (esbuild client bundle + tsc server compile to `dist/`),
  `test` (`vitest run`). Pin `express ^4`, `better-sqlite3 ^11`,
  `esbuild ^0.24`, `vitest ^2`, `supertest`, `jsdom`, `typescript ^5`,
  `@types/express`, `@types/node`, `@types/supertest`. Type `module`.
- `tsconfig.json` — Node ≥ 20 target, ES2022, `module: ESNext`,
  `moduleResolution: bundler`, strict, `outDir: dist`, includes
  `src/**/*` and `tests/**/*`.
- `esbuild.config.mjs` — bundles `src/web/main.ts` → `public/bundle.js`,
  IIFE format, sourcemap, target `es2022`.
- `vitest.config.ts` — per-file environment selection: default `node`,
  `tests/web.test.ts` uses `jsdom`.
- `.gitignore` — `node_modules`, `dist`, `public/bundle.js`,
  `bookmarks.sqlite`, `*.sqlite-journal`, `*.sqlite-wal`,
  `*.sqlite-shm`.

Out:
- Any source code (db/repo/route/web).
- Any test code.
- Running tests against this task (there is nothing to test yet); the
  smoke gate that `npm test` *runs* (even with zero tests) is the only
  assertion.

## Acceptance gates (behaviour-level)

- `npm install` inside `./app/` succeeds.
- `npm test` exits `0` against a zero-test repo (Vitest's "no tests"
  exit-code behaviour, configured to pass under `--passWithNoTests`
  if needed for this bootstrap task).
- `npx tsc --noEmit` succeeds.

## Test sketch

No behavioural tests. The gate is "the harness boots". Build verifies by
running `npm test` and `npx tsc --noEmit` from `./app/`.

## Notes
