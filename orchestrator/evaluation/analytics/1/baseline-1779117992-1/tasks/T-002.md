# T-002 — DB open + schema migration + bookmarks repo

## Goal

Land the persistence half of the three mutating stories. Open
`better-sqlite3`, run the idempotent migration, and expose the three
repo functions `listBookmarks`, `createBookmark`, `deleteBookmark` with
the typed error classes from `design.md`.

## Scope

In:
- `src/db.ts` — `openDb(path)` returns a `better-sqlite3` `Database`,
  applies pragmas (`journal_mode = WAL`, `foreign_keys = ON`,
  `synchronous = NORMAL`), runs `CREATE TABLE IF NOT EXISTS bookmarks
  (...)` and the two indexes. Export `SCHEMA_VERSION = 1`.
- `src/repo/bookmarks-repo.ts` — implements `listBookmarks`,
  `createBookmark`, `deleteBookmark`. Catches
  `SQLITE_CONSTRAINT_UNIQUE` from the URL index and rethrows
  `DuplicateUrlError`. Throws `NotFoundError` when
  `deleteBookmark` affects 0 rows.
- `src/shared/types.ts` — `Bookmark`, `NewBookmark`, `ApiError` DTOs as
  declared in `design.md § Server function signatures`.
- `tests/repo.test.ts` — exercises every repo path against `:memory:`.

Out:
- HTTP layer (T-003 / T-004 / T-005 / T-007).
- Any client code.

## Acceptance gates (behaviour-level)

- Repo unit tests pass (see sketch).
- `npx tsc --noEmit` succeeds.

## Test sketch (behaviour from US-001 / US-002 / US-004 EARS clauses)

- US-001 AC 1: `createBookmark({ title, url })` returns a `Bookmark`
  whose `id` is a positive integer, `created_at` is ISO-8601 UTC, and
  `listBookmarks` returns it.
- US-001 AC 2: a second `createBookmark` with the same URL throws an
  instance of `DuplicateUrlError`; `listBookmarks` still returns
  exactly one row.
- US-002 AC 1: three inserts with increasing timestamps come back in
  newest-first order; two inserts with identical `created_at` come
  back ordered by `id DESC`.
- US-002 AC 3: `listBookmarks` on an empty DB returns `[]` (the route
  layer turns this into the empty-state cue; the repo only asserts the
  shape).
- US-004 AC 1: `deleteBookmark(id)` removes the row;
  `listBookmarks` no longer contains it.
- US-004 AC 2: `deleteBookmark(99999)` against an empty / mismatched
  DB throws `NotFoundError`.

## Notes

- The repo MUST be the only module that touches `Database`. Route
  handlers receive `Bookmark` rows, not raw SQLite outputs (ADR-001).
- No URL normalisation (ADR-003). Store URLs exactly as passed in
  after trimming. Trimming itself happens at the route layer —
  document this in code comments.
