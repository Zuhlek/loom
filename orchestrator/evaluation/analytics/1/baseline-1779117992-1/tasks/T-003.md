# T-003 — Express app shell + static serving + error envelope

## Goal

Stand up the runnable server skeleton and the static HTML/CSS surface so
later behavioural slices can plug in. No bookmark behaviour yet — this
task lands the wiring that every later route + every web test relies on,
including the central JSON error handler shape.

This task has no `satisfies-stories`. Justification: the work is generic
HTTP plumbing (process bootstrap, static middleware, error envelope,
404 fallthrough) that does not deliver an EARS clause from `US-001..
US-004` on its own. Folding it into US-002 (the first GET-shaped story)
would hide cross-cutting wiring behind a list-shaped task and force
T-004 / T-005 / T-007 to re-do scaffolding. Splitting it out keeps each
behavioural slice focused on its story's acceptance criteria.

## Scope

In:
- `src/server.ts` — process bootstrap: `openDb('./bookmarks.sqlite')` →
  `createApp(db)` → `app.listen(3000)`. Logs a single startup line; on
  `EADDRINUSE` exits 1 with a human message (no port search).
- `src/app.ts` — `createApp(db)` builds an Express instance: JSON body
  parser, static middleware for `src/web/` and `public/`, mounts a
  placeholder router at `/api/bookmarks` (the four routes are added in
  T-004 / T-005 / T-007), `GET /` returns `index.html`, central JSON
  error handler that emits `{ error: { code, message } }`, catch-all
  `404 not_found`.
- `src/web/index.html` — static shell: form (title input, url input,
  submit button, two field-error containers), list container, empty-
  state container, top-of-page error banner container, `<script
  type="module" src="/bundle.js">`.
- `src/web/styles.css` — plain CSS for the shell. System fonts only.
  Optional `@media (prefers-color-scheme: dark)` rules permitted (no
  toggle).
- `tests/api.test.ts` — boot-shape smoke (the gate Build relies on
  before later tasks run).

Out:
- Bookmark routes (T-004 / T-005 / T-007).
- Any web TypeScript (`main.ts` is owned by T-005 onward).

## Acceptance gates (behaviour-level)

- Boot smoke: `createApp(db)` against an in-memory DB returns an
  Express instance; supertest `GET /api/bookmarks` returns
  `200` and a JSON body (empty array once T-005 lands; for T-003 the
  placeholder router returns `200 []`).
- `GET /` returns `200`, `Content-Type: text/html`.
- `GET /does-not-exist` returns `404` with the error envelope shape.
- A handler that throws is caught by the central error handler and
  produces `500 internal_error` with the envelope (assert via a
  test-only `/api/__throw` route that is wired only in test builds, or
  by spying on `next(err)` — implementation-choice for Build).

## Test sketch

- `GET /` → `200` + html.
- `GET /api/bookmarks` → `200` + JSON (empty array via placeholder).
- `GET /not-a-thing` → `404` + envelope `{ error: { code:
  'not_found', message: <non-empty string> } }`.
- Forced-throw handler → `500` + envelope `{ error: { code:
  'internal_error', ... } }`.

## Notes

- ADR-002: the DB handle lives on `app.locals.db`. `createApp` takes
  `db` as an argument so tests can pass an `:memory:` handle.
- ADR-007: no CORS middleware. Same-origin only.
- Static-serve order matters: `/bundle.js` is in `public/`, `index.html`
  and `styles.css` are in `src/web/`. Both directories MUST be mounted
  by static middleware so the smoke gate passes after T-001 produces a
  bundle.
