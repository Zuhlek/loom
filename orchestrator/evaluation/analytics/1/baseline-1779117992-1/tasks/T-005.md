# T-005 — List bookmarks end-to-end

## Goal

Deliver US-002 ("See all saved bookmarks in one list") as one vertical
slice: `GET /api/bookmarks` on the server, initial fetch + `renderList`
on the client, including the empty-state branch.

## Scope

In:
- `src/routes/bookmarks.ts` — add `GET /` that calls
  `listBookmarks(db)` and returns `200` with the array. No
  pagination, no query params.
- `src/web/main.ts` — add:
  - `fetchBookmarks()` helper.
  - `renderList(items)` that writes the visible list using
    `textContent` / attribute setters only (no `innerHTML` for user
    data — security envelope) and renders an empty-state element when
    `items.length === 0`.
  - Initial-load wiring: on `DOMContentLoaded`, call `fetchBookmarks`
    then `renderList` with the result.
- `tests/api.test.ts` — GET cases.
- `tests/web.test.ts` — renderList + empty-state + structural absence
  of filter/search/sort controls (US-002 AC 2).

Out:
- The save / open / delete affordances on the rendered row (T-004 /
  T-006 / T-007).

## Acceptance gates (behaviour-level)

US-002 AC 1, AC 2, AC 3.

## Test sketch

API layer (`tests/api.test.ts`):
- `GET /api/bookmarks` against an empty DB → `200 []`.
- After inserting three rows (via the repo directly in test setup),
  `GET` returns them in newest-first order; assert the array's
  `created_at` values are non-increasing and ties are broken by `id`
  descending.

Web layer (`tests/web.test.ts`):
- `renderList([])` makes the empty-state element visible and does not
  emit any `<li>` / row elements (US-002 AC 3).
- `renderList([{ id: 1, title: 'A', url: 'https://a.test',
  created_at: '...' }, ...])` renders one row per item, in the same
  order as the input.
- The page exposes no `input[type=search]`, no element matching
  `[data-role=sort-control]`, and the script never installs a
  client-side filter (US-002 AC 2 — structural).
- A `Bookmark` whose `title` contains `<script>alert(1)</script>`
  renders the literal characters in the DOM (proves `textContent` is
  used, not `innerHTML`).
