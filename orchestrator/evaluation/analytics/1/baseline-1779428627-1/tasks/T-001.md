# T-001 — Scaffold app workspace

## Why this task exists

Every other task assumes the `app/` workspace exists with TypeScript,
Vitest, Express, `better-sqlite3`, and `esbuild` installed and wired
together. This is a single-layer scaffolding task because the tooling
contract (npm scripts, tsconfig paths, vitest config) is consumed by all
nine downstream tasks; replicating it inline in each task would create
contract drift.

## Acceptance criteria (envelope)

1. `app/package.json` declares `scripts.start` (build client + run
   server) and `scripts.test` (Vitest), both runnable from `app/`.
2. Dependencies pinned: `express ^4`, `better-sqlite3` (latest stable),
   `vitest`, `typescript`, `esbuild`, `tsx`, `@types/express`,
   `@types/node`.
3. `tsconfig.json` targets ES2022 for the server source; `tsconfig.client.json`
   extends with `lib: ["DOM", "ES2022"]` for the client bundle.
4. `vitest.config.ts` sets `environment: "jsdom"` for client specs and
   leaves the default for server specs (or configures per-project).
5. `scripts/build-client.ts` invokes `esbuild.build` for
   `src/client/main.ts` and emits to `dist/client/main.js`.
6. `app/.gitignore` excludes `node_modules/`, `dist/`, `data/`.
7. `npx tsc --noEmit -p tsconfig.json` and
   `npx tsc --noEmit -p tsconfig.client.json` exit zero with the empty
   source tree (or `.gitkeep`-anchored directories).

## Test sketch

Envelope task — no behaviour specs of its own. A trivial smoke test
under `test/smoke.test.ts` may assert `1 + 1 === 2` to prove Vitest is
wired; otherwise verification is the typecheck and the existence of the
declared scripts.

## Workspace constraint

All files created by this task MUST live under
`.loom/baseline-1779428627-1/app/`. No file is created outside that
directory.
