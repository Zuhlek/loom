# T-005 â€” Static HTML shell + CSS

## Why this task exists

The HTML shell defines the DOM hook contract (`#bookmark-form`,
`#bookmark-list`, `#empty-state`, etc.) that T-006/T-007/T-008 bind to.
Single-layer (dom/static) is justified because changing a hook id would
need coordinated updates across three behaviour tasks; centralising the
contract here keeps it stable.

## Behaviour delivered

- `public/index.html` ships with the create form (`#bookmark-form`,
  `#bookmark-title`, `#bookmark-url`, `#bookmark-submit`,
  `#form-error`), the list container (`#bookmark-list`), and the
  empty-state node (`#empty-state`, default visible until JS toggles it
  off).
- The shell references `/assets/main.js` (esbuild output, see T-009)
  and `/assets/styles.css`.
- `public/styles.css` is hand-written, includes a
  `@media (prefers-color-scheme: dark)` block (CSS-only respect for the
  system theme; no JS toggle).

## Acceptance criteria (story-anchored)

- US-002 AC3: `#empty-state` exists as a distinct node that the client
  can show/hide; the document does not render a bare empty `<ul>` to
  the user.
- US-003 AC1, AC2: this task does not render rows but ensures the
  static template documents the intended `<a target="_blank"
  rel="noopener noreferrer">` shape (e.g. as a comment) so T-006's
  renderer matches the contract. (The runtime assertion lives in T-006.)

## Test sketch

`test/client/shell.test.ts` (Vitest + jsdom; loads `public/index.html`
via `fs.readFileSync` and parses into the DOM):

- Document contains all required hook ids:
  `#bookmark-form`, `#bookmark-title`, `#bookmark-url`,
  `#bookmark-submit`, `#form-error`, `#bookmark-list`, `#empty-state`.
- `#bookmark-list` is a `<ul>` and is initially empty.
- `#form-error` exists and is initially empty / hidden (e.g.
  `hidden` attribute or empty `textContent`).
- `#empty-state` is initially present (without the `hidden` attribute);
  it will be hidden by the client when the list is non-empty.
