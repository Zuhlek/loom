# T-009 — Server entry + build wiring

## Why this task exists

Stitches the deliverable into a runnable artifact: `npm start` builds
the client bundle, opens the SQLite file at `app/data/bookmarks.db`,
runs the migration, constructs the Express app, and binds
`http://localhost:3000`. Without this, none of the user-facing stories
are reachable through a real browser.

## Behaviour delivered

- `src/server/index.ts`:
  1. Ensures `app/data/` exists (`fs.mkdirSync(..., { recursive: true })`).
  2. Calls `openDatabase('data/bookmarks.db')` → `{ repo, close }`.
  3. Constructs `createApp(repo)`.
  4. Adds `express.static('public')` for the HTML shell.
  5. Adds `express.static('dist/client', { fallthrough: true })` mounted
     at `/assets`.
  6. Listens on `0.0.0.0:3000`, logs "Bookmarks listening on
     http://localhost:3000" to stdout.
  7. On `SIGINT` / `SIGTERM`: closes the DB and exits.
- `scripts/build-client.ts` is finalised so that running
  `tsx scripts/build-client.ts` produces `dist/client/main.js` and
  copies `public/styles.css` to `dist/client/styles.css` (or the static
  mount serves it directly from `public/` — Build picks one and is
  consistent with the HTML shell's `<link>`).
- `package.json` scripts:
  - `"build": "tsx scripts/build-client.ts && tsc -p tsconfig.json"`
  - `"start": "npm run build && node dist/server/index.js"`
  - `"test": "vitest run"`

## Acceptance criteria (envelope, per spec § Constraints › Runtime and origin)

- Running `npm start` from `app/` builds the client, starts the
  server, and the process listens on port 3000.
- A `GET /` returns the HTML shell from `public/index.html`.
- A `GET /assets/main.js` returns the built client bundle with
  `Content-Type: application/javascript`.
- A `GET /api/bookmarks` returns `200` and a JSON array.
- The server makes no outbound network calls during startup (spec §
  Runtime and origin: "no outbound network calls at runtime").
- Coverage of US-001..US-004 is transitive: every story's user-visible
  surface is only reachable because this task wires the process.

## Test sketch

`test/server/smoke.test.ts` (Vitest):

- Boots `createApp(openDatabase(':memory:').repo)` on an ephemeral port
  (`server.listen(0)`).
- `fetch(url + '/api/bookmarks')` → `200`, `[]`.
- `fetch(url + '/api/bookmarks', { method: 'POST', body: JSON.stringify({ title: 'A', url: 'https://a' }) })`
  → `201`.
- A second `fetch` of `/api/bookmarks` → contains the row.
- Closes the server in `afterAll`.

(Note: this smoke test does NOT exercise the static `public/` mount —
those assertions are easier as integration when Build runs `npm start`
locally and the human verifies in a browser per spec § What we're
building.)

## Notes

- The server's `listen` host MUST be `0.0.0.0` or `127.0.0.1`; the spec
  says `http://localhost:3000`, which both resolve to. Picking
  `0.0.0.0` lets a non-localhost host header from the same machine
  still reach the server, which matters for some local dev setups.
- This task does NOT add deployment artifacts (Dockerfile, systemd,
  PM2 config) — explicitly out of scope per spec § Out of scope.
