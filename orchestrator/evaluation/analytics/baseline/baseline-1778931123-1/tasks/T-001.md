# T-001 — Workspace scaffold + dependencies + scripts

## Goal

Stand up the `app/` workspace so that `npm install` resolves the locked stack
(express, better-sqlite3, esbuild, vitest, supertest, tsx, typescript, types),
and so that `npm start` / `npm test` are wired to the build + serve and the
Vitest scripts respectively. Sets the foundation every other task builds on.

## Scope

- `app/package.json` with:
  - `"type": "module"` (or commonjs — chosen to match `tsx`/`better-sqlite3`
    interop; pick whatever lets the rest of the design compile without
    extra ceremony).
  - `scripts.start` runs `tsx scripts/build-client.ts && tsx src/server.ts`.
  - `scripts.test` runs `vitest run`.
  - dependencies as listed in `design.md ## Integration points`.
- `app/tsconfig.json` with strict mode, ES2022 target, module resolution
  matching the chosen module system, includes `src/**/*` and `test/**/*`.
- `app/.gitignore` covering `node_modules/`, `dist/`, `*.sqlite`,
  `*.sqlite-journal`.
- Minimal `app/README.md` documenting `npm install`, `npm start`,
  `npm test` (US-005 docs ergonomics — optional but cheap).

Workspace isolation: every file written here lives under `app/`. No file at
the repo root, no file in sibling workspaces.

## Behaviour delivered

- Satisfies the boot-and-test scaffolding required by US-005 AC1 and AC2;
  the actual server boot lives in T-004 and the Vitest gate is exercised by
  every later task that adds tests.

## Acceptance criteria (derived from satisfied stories)

- US-005 AC1: `npm start` script exists and chains build-client then server
  (the actual handlers come in T-004/T-005). The chain is validated end-to-end
  in T-009.
- US-005 AC2: `npm test` script invokes `vitest run` (non-zero exit on
  failure is Vitest's default).

## Test sketches

- `npm install` from `app/` exits 0; resolves the locked dependency set.
- `node -e "require('./package.json').scripts.start"` prints the build+serve
  chain.
- `node -e "require('./package.json').scripts.test"` prints `vitest run`.

## Notes

- Use `better-sqlite3` known-good version for Node 20; document any prebuilt
  install warnings in README.
- Do not add any framework or runtime not listed in `design.md`.
