# T-004 — Server entrypoint and boot sequence

## Goal

Wire the full process: open DB, run migrations, build the app, mount the
static handler for `dist/client/`, listen on port 3000, install
SIGINT/SIGTERM cleanup. This is what `npm start` ultimately invokes (after
the client build script in T-005). The static handler points at
`dist/client/` and serves `index.html` at `/` and `main.js`/`styles.css` at
`/static/*`.

## Scope

- `app/src/server.ts`:
  - `startServer(opts?: { port?, dbPath? })` →
    1. `openDb(dbPath ?? 'app/bookmarks.sqlite')`.
    2. `migrate(db)`.
    3. `makeRepo(db)`.
    4. `buildApp(repo)` plus a static handler mounted at `/` (serves
       `index.html`) and `/static/*` (serves bundled assets), pointing to
       `dist/client/`.
    5. `app.listen(port ?? 3000, '127.0.0.1' /* loopback only */)`.
    6. Install SIGINT/SIGTERM → `server.close()` + `db.close()`.
  - Log `listening on http://localhost:3000` on successful listen.
  - On `openDb` / `migrate` failure: log + `process.exit(1)`.
- `app/test/server-boot.test.ts` — uses `startServer({ port: 0, dbPath: ':memory:' })`
  to assert the server can start, GET `/` returns `200`, GET `/api/bookmarks`
  returns `200 []`, and `close()` shuts down cleanly.

## Behaviour delivered

- The same-origin boot that US-005 AC1 demands.
- The HTML shell-serving path that US-002 AC1 relies on so the user can
  load `http://localhost:3000` and see the list page.

## Acceptance criteria (derived from satisfied stories)

- US-002 AC1: `GET /` returns the rendered `index.html` (with the bundle
  script tag pointing at `/static/main.js`); `GET /api/bookmarks` returns
  the list.
- US-005 AC1: `npm start` (after the build script runs) results in a server
  listening on `http://localhost:3000` and serving both UI and API on the
  same origin.

## Test sketches

- `startServer({ port: 0, dbPath: ':memory:' })` resolves; the returned
  `Server` has a non-zero `address().port`.
- `GET /` → `200`, `Content-Type: text/html`.
- `GET /api/bookmarks` → `200 []`.
- `GET /static/main.js` → `200` (uses a fixture or pre-built bundle in test
  setup; or asserts the route is wired by checking for `404` only when the
  file truly is missing).
- SIGINT handler closes the server (verified by sending the signal in a
  child process test or by calling the handler function directly).

## Notes

- Listen on `127.0.0.1` only — never `0.0.0.0` — per the local-only
  constraint in `spec.md`.
- The static handler must serve `index.html` at the root and `main.js` /
  `styles.css` under `/static/*` so the bundle URL is stable.
