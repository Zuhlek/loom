# T-006 — Client API module and list rendering

## Goal

Implement the client-side fetch wrapper (`listBookmarks`, `createBookmark`,
`deleteBookmark`) and a pure `render(list, root)` function that produces the
list DOM (with empty-state branch and link safety). Wire `main.ts` so that
on page load it fetches and renders the list. Anchors are emitted with
`target="_blank"` and `rel="noopener"` (US-003).

## Scope

- `app/src/client/api.ts`:
  - `listBookmarks(): Promise<Bookmark[]>` (GET `/api/bookmarks`).
  - `createBookmark(input): Promise<Bookmark>` (POST `/api/bookmarks`).
  - `deleteBookmark(id): Promise<void>` (DELETE `/api/bookmarks/:id`).
  - On non-2xx, parse the error envelope and `throw { code, message, field? }`.
- `app/src/client/render.ts`:
  - `render(bookmarks: Bookmark[], root: HTMLElement): void` — full
    re-render (no diff). Empty list → renders an empty-state message
    (e.g. `<p class="empty">No bookmarks yet. Save your first one above.</p>`).
    Populated → renders `<ul>` of rows, each containing:
      - `<a href={url} target="_blank" rel="noopener">{title}</a>`
      - `<span class="url">{url}</span>`
      - a delete button (logic in T-008; render emits the element with a
        stable selector like `data-action="delete"` and `data-id={id}`).
  - All text/attribute writes use `textContent` / `setAttribute` — no
    `innerHTML` with user data.
- `app/src/client/main.ts`:
  - `bootstrap()` — calls `listBookmarks()` and `render(list, root)` on
    load; exposes a `refresh()` for T-007 / T-008 to invoke after mutation.
- `app/test/client-render.test.ts` — jsdom tests for `render`.

## Behaviour delivered

- US-002 AC1, AC2, AC3 (list rendering + empty state) and the entirety
  of US-003 (link target + rel).
- POST/DELETE wiring used by T-007 / T-008.

## Acceptance criteria (derived from satisfied stories)

- US-002 AC1: render produces a `<ul>` whose children appear in the same
  order as the input array (server returns newest-first per T-003).
- US-002 AC2: every row contains both the title text and the URL string
  as DOM text.
- US-002 AC3: when `bookmarks.length === 0`, render emits the empty-state
  message and does NOT emit an empty `<ul>` element.
- US-003 AC1: every row's primary link has `target="_blank"`.
- US-003 AC2: every row's primary link has `rel="noopener"`.

## Test sketches

- `render([], root)` → `root.querySelector('ul')` is `null`; the
  empty-state element is present.
- `render([{id:1, url:'https://a/', title:'A', created_at:...}], root)`
  → exactly one `<li>`; its `<a>` has `href === 'https://a/'`,
  `target === '_blank'`, `rel === 'noopener'`; row text contains `'A'`
  and `'https://a/'`.
- Ordering: `render([b2, b1], root)` (b2 newer) → first `<li>` is b2.
- Link safety: a title or URL containing `<script>` does not produce a
  script tag in the DOM (assert via `root.querySelectorAll('script')`).
- `listBookmarks()`: stub `globalThis.fetch` to return `{ok: true,
  json: () => [...] }` and assert the result. Stub returns `{ok: false,
  status: 500, json: () => ({error:{code:'internal',message:'x'}})}` and
  assert the wrapper throws an object with `code === 'internal'`.

## Notes

- `render` is pure (input list, output DOM). Easy to unit test in jsdom.
- The delete button selector and `data-id` attribute are the integration
  seam T-008 uses.
