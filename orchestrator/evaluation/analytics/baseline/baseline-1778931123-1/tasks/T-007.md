# T-007 — Save-bookmark form

## Goal

Hook the save form: on submit, call `createBookmark({url, title})` from the
client API; on success, clear the form and refetch the list (reload-on-write
per ADR-005); on `400`/`409`, render the inline error under the offending
field with the message from the server envelope. No optimistic UI.

## Scope

- `app/src/client/form.ts`:
  - `attachSaveForm(formEl, refresh: () => Promise<void>): void`.
  - Listens for `submit`; calls `preventDefault`.
  - Reads `url` and `title` from inputs; calls `createBookmark`.
  - On success: reset form inputs; clear any inline errors; await
    `refresh()`.
  - On thrown `{code, message, field}`: render the error under the field
    (`field === 'url'` → URL field error; `field === 'title'` → title
    field error; no field → general form error slot).
- `app/src/client/main.ts`:
  - Wire `attachSaveForm` to `#save-form` and to `bootstrap.refresh`.
- `app/test/client-form.test.ts` — jsdom tests with `fetch` stubbed.

## Behaviour delivered

- US-001 AC1, AC2, AC3, AC4 — entire save story at the client edge.

## Acceptance criteria (derived from satisfied stories)

- US-001 AC1: submitting valid `{url, title}` calls `createBookmark`,
  clears the form, and triggers a list refresh that places the new
  bookmark at the top.
- US-001 AC2: when `createBookmark` rejects with
  `{code: 'duplicate_url', field: 'url'}`, the URL field's inline error
  element contains a message naming the duplicate URL.
- US-001 AC3: when the URL field is empty or fails parsing, the inline
  error renders the validation message under the URL field. (Server is the
  source of truth — submit blocks until response; no client-side URL
  parser. Or: optional preflight that mirrors server checks.)
- US-001 AC4: when the title is empty or whitespace-only, the inline error
  renders under the title field.

## Test sketches

- Stub `fetch` to return `201 {id:1,...}`; submit form; assert
  `fetch` was called with `POST /api/bookmarks` and JSON body; assert
  inputs are cleared; assert `refresh` was awaited.
- Stub `fetch` to return `409 {error:{code:'duplicate_url',field:'url',message:'…'}}`;
  submit; assert the URL error slot contains the server message.
- Stub `fetch` to return `400 {error:{code:'validation',field:'url',message:'…'}}`;
  assert the URL error slot contains the message.
- Stub `fetch` to return `400 {error:{code:'validation',field:'title',message:'…'}}`;
  assert the title error slot contains the message.
- After a successful save, error slots are cleared on the next submit.

## Notes

- Error rendering uses `textContent` — no HTML injection.
- No optimistic UI; await the POST response, then refetch.
