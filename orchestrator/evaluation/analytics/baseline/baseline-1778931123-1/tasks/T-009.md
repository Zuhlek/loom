# T-009 — End-to-end smoke and persistence gate

## Goal

Prove that the assembled system boots, serves, persists, and tests cleanly
with the two locked commands. Verifies the persistence-across-restart
constraint and that `npm test` exits non-zero on failure. Acts as the
final acceptance gate for the project.

## Scope

- `app/test/smoke.test.ts` (or a shell script invoked from `package.json`):
  - Boots `startServer({ port: <ephemeral>, dbPath: <temp> })` in-process.
  - GETs `/` → asserts `200` and HTML content.
  - GETs `/api/bookmarks` → asserts `200 []`.
  - POSTs a valid bookmark → asserts `201`.
  - Closes the server.
  - Calls `startServer` again with the SAME `dbPath` → asserts `GET
    /api/bookmarks` now returns the bookmark from before the restart.
  - Closes and removes the temp DB file.
- Workspace-isolation check: after `npm test`, no new files appear outside
  `app/`. Implementable as a small bash diff against `git status` of the
  repo root before/after the suite.
- A deliberately-failing test (committed temporarily during the gate
  dry-run, then removed before final delivery) to confirm `npm test` exits
  non-zero. Alternatively: assert via `child_process.spawnSync('npm', ['test'])`
  in a meta-test that's only run when an env var is set; not required for
  shipping.

## Behaviour delivered

- US-005 AC1 — `npm start` boots the whole app and serves UI + API on
  `http://localhost:3000`.
- US-005 AC2 — `npm test` runs the full Vitest suite and exits non-zero
  on failure.
- Persistence-across-restart constraint from `spec.md ## Constraints`.

## Acceptance criteria (derived from satisfied stories)

- US-005 AC1: a fresh checkout running `npm install && npm start` from
  `app/` produces a process that responds to `GET /` with the HTML page
  and `GET /api/bookmarks` with `200 []` within a small timeout.
- US-005 AC2: `npm test` from `app/` reports the full suite and exits 0
  on green / non-zero on any failure.

## Test sketches

- Restart smoke: post a bookmark, close server, reopen with same
  `dbPath`, assert the bookmark is returned.
- Workspace isolation: `git status` from repo root shows no new files
  outside `app/` after `npm start` and `npm test`.
- Boot port: assert the server binds `127.0.0.1` (loopback only) — read
  `server.address()` and check `address === '127.0.0.1'`.
- `npm start` chain: assert the script runs `build-client.ts` before
  `server.ts` (verifiable in T-001's `package.json` test if not here).
