# T-001 — Bootstrap app/ project skeleton and npm scripts

## Why a single-layer (tooling) task is justified

Every downstream task (T-002..T-005) requires a working `npm install`, a
TypeScript compiler with the right `lib`s for server vs. browser, a Vitest
runner, and an `esbuild`-driven client build script. Without this scaffold,
no observable-behaviour task can compile its code or run its tests. This task
ships **no user-facing behaviour** but is the smallest possible enabling
slice — it does not include any production source code.

## Goal

Stand up the `app/` workspace so subsequent tasks can `npm install`, `npm
test`, `npm run build:client`, and `npm start` without further config churn.

## Deliverables

- `app/package.json` with:
  - `"private": true`
  - dependencies: `express` ^4, `better-sqlite3` ^11
  - devDependencies: `typescript` ^5, `vitest` ^1, `supertest` ^7, `esbuild`
    ^0.20, `tsx` ^4, `@types/express`, `@types/better-sqlite3`,
    `@types/supertest`, `@types/node`
  - scripts:
    - `"build:client": "tsx scripts/build-client.ts"`
    - `"build": "npm run build:client"`
    - `"prestart": "npm run build:client"`
    - `"start": "tsx src/server/index.ts"`
    - `"test": "vitest run"`
- `app/tsconfig.json` — NodeNext, strict, `target: ES2022`, `lib: ["ES2022"]`,
  `outDir: dist`, includes `src/server/**`, `tests/**`, `scripts/**`.
- `app/tsconfig.client.json` — extends base; `lib: ["ES2022","DOM"]`,
  includes only `src/client/**`.
- `app/vitest.config.ts` — `environment: "node"`, `include:
  ["tests/**/*.test.ts"]`.
- `app/.gitignore` — `node_modules/`, `dist/`, `bookmarks.db`.

## Acceptance criteria

1. `cd app && npm install` succeeds with the pinned stack.
2. `npm test` exits 0 (no tests yet; Vitest reports "no test files found"
   should be accepted by configuring `passWithNoTests: true` OR the suite is
   left to pass once T-002 lands — Build's choice; the script wiring is the
   deliverable here).
3. `npm run build:client` is wired (the script file itself is delivered by
   T-004; this task only ensures the npm script exists).
4. No file is written outside `.loom/baseline-1778963742-1/app/`.

## Test sketch

No behaviour tests in this task. Verification is `npm install` exiting 0 and
`npm test` not erroring on missing config.
