# T-002 — DB layer and bookmarks repository

## Goal

Deliver the persistence surface (`openDb`, `createBookmarksRepo`) and pure
validation helpers, with Vitest unit tests against an in-memory SQLite that
prove insert, list, delete, and duplicate-rejection behave per `design.md`.

## Deliverables

- `app/src/server/db.ts` — `openDb(filename: string): Database` that runs the
  `CREATE TABLE IF NOT EXISTS bookmarks (...)` and the
  `CREATE INDEX IF NOT EXISTS idx_bookmarks_created_at` from
  `design.md` §Data model before returning the handle.
- `app/src/server/validation.ts` — `validateUrl(raw)`, `validateTitle(raw)`
  returning the discriminated unions from `design.md` §Module exports.
  `validateUrl` MUST attempt `new URL(raw)` and reject on throw or empty.
- `app/src/server/bookmarks.ts` — `Bookmark` type, `CreateInput` type,
  `DuplicateUrlError` class, `createBookmarksRepo(db)` returning
  `{ insert, list, deleteById }`:
  - `insert` canonicalises via `new URL(input.url).toString()` per ADR-009,
    stamps `created_at = Date.now()`, and translates the SQLite UNIQUE
    violation into `DuplicateUrlError`.
  - `list` returns rows `ORDER BY created_at DESC, id DESC` per ADR-010.
  - `deleteById` returns `false` when `info.changes === 0`, `true` otherwise.
- `app/tests/validation.test.ts` — unit tests for both validators (empty,
  non-string, malformed, well-formed).
- `app/tests/repo.test.ts` — unit tests using `new Database(':memory:')`:
  insert returns a populated row; list is newest-first with `id DESC`
  tiebreak; deleteById returns true/false correctly; duplicate insert throws
  `DuplicateUrlError`; URL normalisation collapses `https://example.com` and
  `https://example.com/`.

## Acceptance criteria (story-linked)

- **US-001 AC1** — `repo.insert` persists a row with id, url, title,
  `created_at` populated; covered by `repo.test.ts`.
- **US-001 AC3** — duplicate `url` (incl. trailing-slash variant) throws
  `DuplicateUrlError`; covered by `repo.test.ts`.
- **US-001 AC4** — `validateUrl` / `validateTitle` reject empty and malformed
  inputs; covered by `validation.test.ts`.
- **US-002 AC2** — `repo.list()` returns rows newest-first with a deterministic
  `id DESC` tiebreaker; covered by `repo.test.ts`.
- **US-004 AC1** — `repo.deleteById(id)` removes the row; subsequent
  `repo.list()` omits it; covered by `repo.test.ts`.
- **US-004 AC3** — `repo.deleteById(unknownId)` returns `false`; covered by
  `repo.test.ts`.

## Test sketch

```ts
// repo.test.ts
const db = openDb(':memory:');                       // openDb accepts ':memory:'
const repo = createBookmarksRepo(db);
const a = repo.insert({ url: 'https://example.com', title: 'A' });
expect(a.id).toBeGreaterThan(0);
expect(a.created_at).toBeGreaterThan(0);

expect(() => repo.insert({ url: 'https://example.com/', title: 'dup' }))
  .toThrow(DuplicateUrlError);                       // ADR-009 normalisation

const b = repo.insert({ url: 'https://b.example', title: 'B' });
expect(repo.list().map(r => r.id)).toEqual([b.id, a.id]); // newest first, id tiebreak

expect(repo.deleteById(a.id)).toBe(true);
expect(repo.deleteById(a.id)).toBe(false);
```

## Notes

- `openDb` must accept `':memory:'` so tests can share the same code path as
  production. If the production path resolves the filename to an absolute
  path, branch on the literal `':memory:'` to skip resolution.
- Do NOT touch routes or Express here.
