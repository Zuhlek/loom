# T-003 — Express app factory, routes, supertest integration

## Goal

Build the HTTP surface declared in `design.md` §Interfaces: `createApp(db)`
mounts JSON middleware, the `/api/bookmarks` router, the static UI mounts,
and a final error-handling middleware. Cover all four endpoints and every
documented status code with supertest integration tests against
`createApp(new Database(':memory:'))` per ADR-007.

## Deliverables

- `app/src/server/app.ts` — `createApp(db: Database): express.Express`:
  - `express.json()` body parsing.
  - Mounts the `/api/bookmarks` router from `routes.ts`.
  - Mounts `express.static(path.join(__dirname, '../../dist/client'))` and
    `express.static(path.join(__dirname, '../client'))` both at `/static`
    per ADR-008 (so `/static/main.js` and `/static/style.css` both resolve).
  - Serves `GET /` via `res.sendFile` of `src/client/index.html` (path
    resolved relative to the source layout; OK if `index.html` does not yet
    exist in T-003 — `app.ts` MUST NOT crash at construction time when it
    does not).
  - Final `(err, req, res, next)` error middleware returns
    `500 { error: "internal", message }` and logs the error.
- `app/src/server/routes.ts` — Express router for `/api/bookmarks`:
  - `POST /` — guards body shape (object), runs `validateUrl` and
    `validateTitle`, calls `repo.insert`, returns 201 on success, 400 for
    each validation failure, 409 for `DuplicateUrlError`.
  - `GET /` — `repo.list()` → 200 array.
  - `DELETE /:id` — parses `:id` as positive integer (400 `invalid_id`
    otherwise); calls `repo.deleteById`; 204 on true, 404 `not_found` on
    false.
- `app/tests/api.bookmarks.test.ts` — supertest suite covering every row of
  `design.md` §Server failure modes plus the happy paths.

## Acceptance criteria (story-linked)

- **US-001 AC1** — `POST /api/bookmarks` with valid `{url,title}` → 201 with
  the documented body shape.
- **US-001 AC3** — duplicate URL POST → 409 `duplicate_url`; trailing-slash
  variant collides too (ADR-009).
- **US-001 AC4** — empty body / non-object → 400 `invalid_body`; empty url →
  400 `invalid_url`; empty title → 400 `invalid_title`.
- **US-002 AC1/AC2** — `GET /api/bookmarks` returns all rows in `created_at
  DESC, id DESC` order.
- **US-002 AC3** — empty DB → `[]`.
- **US-004 AC1** — `DELETE /api/bookmarks/:id` → 204 and subsequent `GET`
  omits the row.
- **US-004 AC3** — `DELETE` of unknown id → 404 `not_found`; non-integer id
  → 400 `invalid_id`; further `GET` still 200.

## Test sketch

```ts
const db = openDb(':memory:');
const app = createApp(db);
const agent = request(app);

const created = (await agent.post('/api/bookmarks').send({ url: 'https://a.test', title: 'A' })).body;
expect(created).toMatchObject({ url: 'https://a.test/', title: 'A' });

await agent.post('/api/bookmarks').send({ url: 'https://a.test/', title: 'dup' }).expect(409);
await agent.post('/api/bookmarks').send({ url: '', title: 'x' }).expect(400);
await agent.post('/api/bookmarks').send({ url: 'https://b.test', title: '' }).expect(400);
await agent.post('/api/bookmarks').send('not json').type('text/plain').expect(400);

const list = (await agent.get('/api/bookmarks').expect(200)).body;
expect(list[0].url).toBe(created.url);

await agent.delete(`/api/bookmarks/${created.id}`).expect(204);
await agent.delete(`/api/bookmarks/${created.id}`).expect(404);
await agent.delete('/api/bookmarks/abc').expect(400);
```

## Notes

- Tests must NEVER call `app.listen()`. supertest handles port-less binding.
- `createApp` MUST be safe to construct even when `dist/client/main.js` and
  `src/client/index.html` do not yet exist (those are T-004 deliverables);
  the static mounts may simply 404 until then.
- All failure-mode rows in `design.md` §State and error handling must be
  asserted explicitly.
