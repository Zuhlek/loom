# T-004 — Vanilla-TS client bundle

## Goal

Ship the single-page UI: one HTML page, one CSS file, one TS entry compiled
to `dist/client/main.js` by an `esbuild` driver script. The page lets the
user save, list, open, and delete bookmarks against the JSON API.

## Deliverables

- `app/src/client/index.html` — single page with:
  - `<link rel="stylesheet" href="/static/style.css">`.
  - `<form id="add-form">` with `<input name="url">`, `<input name="title">`,
    a submit button, and an inline `<p id="form-error">` slot.
  - `<section id="list-error">` slot above the list (for the non-fatal banner
    used by US-004 AC3 and the load-failure path).
  - `<ul id="bookmarks">` for the rendered list.
  - `<script type="module" src="/static/main.js"></script>`.
- `app/src/client/style.css` — minimal stylesheet; a `prefers-color-scheme:
  dark` block is permitted (CSS-only dark mode is in scope per spec).
- `app/src/client/main.ts` — vanilla TS entry:
  - Holds the three-field state from `design.md` §Client-side state shape:
    `bookmarks`, `formError`, `listError`.
  - Exports a pure `renderList(bookmarks, hostUl)` helper (or equivalent
    pure-render fn) so the DOM contract is testable.
  - Each list row renders the title as an `<a target="_blank"
    rel="noopener noreferrer" href="<url>">` (US-003 + spec security
    envelope) using `textContent` for the title (no innerHTML, no
    user-controlled HTML injection).
  - Each row carries a delete button wired to `DELETE /api/bookmarks/:id`;
    after success → refetch + re-render; on 404 → set the non-fatal banner
    per US-004 AC3 and still refetch.
  - On form submit: validate non-empty client-side (cheap), POST, on 201
    clear form + refetch + render, on 409 set inline "URL is already saved"
    error, on 400 surface the server message, on network/500 surface
    "Could not save — try again".
  - On load: `listBookmarks()` then `renderList`. On failure, set
    `listError` and render empty.
  - Empty-state: when `bookmarks.length === 0` render
    `<p class="empty">No bookmarks yet.</p>` (US-002 AC3).
- `app/scripts/build-client.ts` — calls
  `esbuild.build({ entryPoints: ['src/client/main.ts'], bundle: true,
   outfile: 'dist/client/main.js', format: 'esm', target: 'es2022',
   sourcemap: true })`. Exits non-zero on build failure.
- `app/tests/client.render.test.ts` — Vitest test that imports `renderList`,
  feeds it a fixture array + a `document.createElement('ul')` (via
  `happy-dom` or `jsdom` — pick the lighter-weight one and add it to
  devDeps under T-001 amendment if needed, OR construct lightweight DOM
  via a single dependency-free `linkedom` import; the simpler choice for
  T-004 is to set `environment: 'happy-dom'` for this file only via
  `// @vitest-environment happy-dom` and add `happy-dom` to devDeps).
  - Asserts each `<a>` has `target="_blank"`, `rel="noopener noreferrer"`,
    and its `textContent` equals the input title (no HTML injection).
  - Asserts empty input renders the empty-state paragraph instead of a list.

## Acceptance criteria (story-linked)

- **US-001 AC2** — successful save re-fetches + re-renders the list with the
  new row at the top, no full page reload (rendering function exercised in
  the DOM unit test; full client wiring exercised by T-005 smoke).
- **US-001 AC3/AC4** — inline `form-error` slot is populated on 409 / 400.
- **US-002 AC1/AC2/AC3** — initial render fetches and lists newest-first;
  empty list renders empty-state paragraph.
- **US-003 AC1/AC2** — each link is `target="_blank"
  rel="noopener noreferrer"`; clicking opens a new tab, leaving the
  bookmarks list intact in the original tab (browser-guaranteed by the
  attributes; asserted in DOM unit test).
- **US-004 AC1/AC2** — delete button triggers `DELETE`; on success re-fetch
  + re-render removes the row.
- **US-004 AC3** — on 404, `list-error` banner is populated and the list is
  re-fetched.

## Test sketch

```ts
// client.render.test.ts (// @vitest-environment happy-dom)
import { renderList } from '../src/client/main';
const ul = document.createElement('ul');
renderList([{ id: 1, url: 'https://a.test/', title: '<b>A</b>', created_at: 1 }], ul);
const a = ul.querySelector('a')!;
expect(a.getAttribute('target')).toBe('_blank');
expect(a.getAttribute('rel')).toBe('noopener noreferrer');
expect(a.textContent).toBe('<b>A</b>');         // textContent, not innerHTML
expect(a.innerHTML).not.toContain('<b>');

const empty = document.createElement('ul');
renderList([], empty);
expect(empty.parentElement?.querySelector('.empty') ?? empty.querySelector('.empty')).toBeTruthy();
```

## Notes

- `renderList` must be exported from `main.ts` (or a sibling pure module
  imported by `main.ts`) so it can be unit-tested in isolation.
- `npm run build:client` must succeed and produce `app/dist/client/main.js`.
- Do not introduce any framework; vanilla DOM APIs only.
