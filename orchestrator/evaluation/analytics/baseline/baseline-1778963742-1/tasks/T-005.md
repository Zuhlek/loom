# T-005 — Server entrypoint + E2E smoke

## Goal

Close the loop: a real `npm start` boots the server on
`http://localhost:3000`, serves UI + API from the same origin, and persists
to `app/bookmarks.db`. An end-to-end smoke test spins the same app on an
ephemeral port and walks save → list → delete to prove all five layers wire
together.

## Deliverables

- `app/src/server/index.ts` — side-effectful boot:
  - `const db = openDb(path.resolve(process.cwd(), 'bookmarks.db'));`
  - `createApp(db).listen(3000, () => console.log('Bookmarks running on http://localhost:3000'));`
  - No exports.
- `app/tests/e2e.smoke.test.ts` — Vitest test:
  - `beforeAll`: build app via `createApp(openDb(':memory:'))`, call
    `server = app.listen(0)`, read `server.address().port`.
  - Test 1 (`GET /`): expects 200 and `text/html` so the UI mount is wired.
  - Test 2 (`GET /static/main.js`): expects 200 and a non-empty body so the
    bundle is reachable (assumes T-004's build was run; the test file runs
    `npm run build:client` in a `beforeAll` if `dist/client/main.js` is
    absent, or skips with a clear message — Build picks; the simpler choice
    is to call the esbuild driver programmatically in `beforeAll`).
  - Test 3 (`POST → GET → DELETE → GET` round-trip via `node:http` against
    `http://localhost:<port>`): the deleted row is absent from the final
    list; the test asserts US-001 AC1, US-002 AC1/AC2, US-004 AC1/AC2 wired
    end-to-end.
  - `afterAll`: `server.close()`.

## Acceptance criteria (story-linked)

All four stories — the E2E smoke exercises the integrated stack:

- **US-001 AC1** — POST returns 201 + body, row is persisted.
- **US-002 AC1** — GET returns the saved row.
- **US-002 AC2** — multi-insert order is newest-first.
- **US-003 AC1** — UI HTML reaches the browser at `/` and references
  `/static/main.js`; the link-attribute assertion lives in T-004's DOM unit
  test (browser tab semantics are not exercisable in a Node test runner).
- **US-004 AC1/AC2** — DELETE returns 204; subsequent GET omits the row.

## Test sketch

```ts
let server: import('http').Server;
let port: number;
beforeAll(async () => {
  // ensure client bundle exists
  await import('../scripts/build-client');         // or run programmatically
  const app = createApp(openDb(':memory:'));
  server = app.listen(0);
  port = (server.address() as import('net').AddressInfo).port;
});
afterAll(() => new Promise(r => server.close(() => r(null))));

it('serves the UI shell at /', async () => {
  const res = await fetchJson(`http://localhost:${port}/`);
  expect(res.status).toBe(200);
  expect(res.headers['content-type']).toMatch(/text\/html/);
});

it('round-trips a bookmark end-to-end', async () => {
  const created = await postJson(`http://localhost:${port}/api/bookmarks`, { url: 'https://a.test', title: 'A' });
  expect(created.status).toBe(201);
  const list1 = await fetchJson(`http://localhost:${port}/api/bookmarks`);
  expect(list1.body[0].url).toBe('https://a.test/');
  const del = await deleteReq(`http://localhost:${port}/api/bookmarks/${created.body.id}`);
  expect(del.status).toBe(204);
  const list2 = await fetchJson(`http://localhost:${port}/api/bookmarks`);
  expect(list2.body).toEqual([]);
});
```

## Notes

- The E2E test must use `app.listen(0)` (ephemeral port) so it does not
  collide with a developer running `npm start` on 3000.
- `index.ts` must resolve `bookmarks.db` relative to `process.cwd()` so the
  workspace-isolation constraint holds (`npm start` is invoked from
  `./app/`).
- Server must shut down cleanly in `afterAll`; leaked listeners will hang
  the Vitest run.
