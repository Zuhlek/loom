## Goal

Lay down the empty workspace tree and the minimal `package.json` /
`tsconfig.json` so every downstream task has a stable place to write files
and a runnable `npm install` baseline.

## Scope

- Create `app/package.json` with:
  - `"type": "module"`
  - `"scripts": { "start": "placeholder", "test": "vitest run", "build": "node build.mjs", "typecheck": "tsc --noEmit" }`
    (the `start` script will be finalised in T-010; placeholder must not error
    in this task's verification, e.g. `node -e "console.log('placeholder')"`.)
  - Dependencies pinned: `express`, `better-sqlite3`.
  - Dev dependencies pinned: `typescript`, `vitest`, `supertest`,
    `@types/express`, `@types/better-sqlite3`, `@types/supertest`,
    `@types/node`, `esbuild`, `tsx`.
- Create `app/tsconfig.json` targeting `ES2022`, `module: ESNext`,
  `moduleResolution: Bundler`, `strict: true`, `noEmit: true`,
  `types: ["node", "vitest/globals"]` as appropriate.
- Create `app/.gitignore` covering `node_modules/`, `public/app.js`,
  `data/*.sqlite*`, build artefacts.
- Create empty directories: `app/src/server/`, `app/src/web/`,
  `app/public/`, `app/tests/`, `app/data/`.
- Create stub `app/build.mjs` that exports an esbuild config function but
  does not yet run a build (final wiring in T-010).

## Out of scope

- Actual source code (deferred to T-002+).
- esbuild execution wiring (T-010).
- Real `start` script (T-010).

## Acceptance

- `npm install` from `app/` succeeds.
- `npx tsc --noEmit` reports no errors against the empty project.
- Directory layout matches `design.md` §Directory layout.

## Verification

Environment: `node-test`.

Commands run from `app/`:
- `npm install`
- `npx tsc --noEmit`
