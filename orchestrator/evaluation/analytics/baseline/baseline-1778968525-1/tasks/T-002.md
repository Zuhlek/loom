## Goal

Implement the SQLite bootstrap module that opens the database file, runs
the idempotent schema migration, and returns a configured `Database`
handle. Enforce the `UNIQUE(url)` invariant declared in `spec.md`
constraints.

## Scope

- `src/server/db/index.ts` exporting:
  - `openDatabase(filename?: string): Database` — opens via `better-sqlite3`;
    defaults to `app/data/bookmarks.sqlite`, ensures the parent directory
    exists, sets `PRAGMA journal_mode = WAL` and `PRAGMA foreign_keys = ON`.
  - `runMigrations(db: Database): void` — runs the idempotent
    `CREATE TABLE IF NOT EXISTS bookmarks ...` and the
    `idx_bookmarks_created_at` index.
- `bookmarks` table columns:
  - `id INTEGER PRIMARY KEY AUTOINCREMENT`
  - `title TEXT NOT NULL`
  - `url TEXT NOT NULL UNIQUE`
  - `created_at TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%fZ','now'))`
- Test helper (exported or test-local) for constructing an in-memory DB
  used by downstream tests.

## Acceptance criteria delivered

- Storage invariant for US-001 AC4 (UNIQUE constraint will raise on
  duplicate URL).
- Storage invariant for US-002 AC1 (`created_at` text suitable for
  chronological ordering).
- Schema present so US-004 AC1 deletes can hit a real table.

## Tests (Vitest)

See `tests.md` § T-002. Notable:

- Opening against `:memory:` and running migrations yields a `bookmarks`
  table.
- Migrations are idempotent.
- A second insert with the same `url` raises `SQLITE_CONSTRAINT_UNIQUE`.
