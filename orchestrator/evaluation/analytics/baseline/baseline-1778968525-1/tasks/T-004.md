## Goal

Implement the pure validation layer for the create-bookmark payload.
Throws `ValidationError` with the API-stable `code` values that routes
will surface (US-001 AC2, AC3).

## Scope

- `src/server/validation.ts`:
  - `validateNewBookmark(raw: unknown): { title: string; url: string }`.
  - Title rule: must be a string, trim, non-empty after trim. Else throw
    `ValidationError({ code: 'INVALID_TITLE', message: 'Title is required' })`.
  - URL rule: must be a string and pass `new URL(value)` without throwing.
    Else throw `ValidationError({ code: 'INVALID_URL', message: 'URL must be a syntactically valid URL' })`.
  - Reject payloads that are not objects (throws as `INVALID_BODY` or
    `INVALID_TITLE` — pick one and document; recommended: throw
    `ValidationError({ code: 'INVALID_TITLE' })` for missing title or
    `INVALID_URL` for missing url; non-object payload returns a single
    `INVALID_TITLE` first).
- `src/server/errors.ts`: ensure `ValidationError` exists with
  `code: 'INVALID_TITLE' | 'INVALID_URL' | 'INVALID_ID'` and a `message`
  field.

## Acceptance criteria delivered

- US-001 AC2 — invalid/missing URL is rejected before any SQLite write.
- US-001 AC3 — empty / whitespace-only title is rejected before any
  SQLite write.
