## Goal

Assemble the Express application factory and the global error-handling
middleware that maps each domain error to the
`{ error: { code, message } }` API contract (ADR-008). The factory takes
the DB handle (ADR-006) so tests can inject per-test DBs.

## Scope

- `src/server/errors.ts` — finalise:
  - `ValidationError(code, message)` (`code`: `'INVALID_TITLE' | 'INVALID_URL' | 'INVALID_ID'`).
  - `DuplicateUrlError(message?)` (`code: 'DUPLICATE_URL'`).
  - `NotFoundError(message?)` (`code: 'NOT_FOUND'`).
  - Helper `httpStatusFor(err)` mapping each class to 400/409/404.
- `src/server/middleware/error.ts` — Express error handler:
  - Maps known classes to their declared status with body shape
    `{ error: { code, message } }`.
  - Handles malformed JSON from `express.json()` (SyntaxError with
    `type === 'entity.parse.failed'`) → 400 / `INVALID_BODY`.
  - Falls back to 500 / `INTERNAL` for unknown errors; logs server-side.
- `src/server/app.ts`:
  - `buildApp(db: Database): express.Express`.
  - Mount `express.json()`, route registration placeholder (real routes
    arrive in T-006), then the error middleware last.
  - No `app.listen` call (that lives in T-007).

## Acceptance criteria delivered

- US-001 AC2/AC3 path infrastructure (400 mapping).
- US-001 AC4 path infrastructure (409 mapping).
- US-004 AC2 path infrastructure (500 mapping when SQLite fails).

## Tests

See `tests.md` § T-005. Each error class maps to the declared HTTP code
and JSON body.
