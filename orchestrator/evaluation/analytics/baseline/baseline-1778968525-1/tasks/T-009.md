## Goal

Implement the vanilla-TS frontend: typed fetch wrappers, DOM render
helpers, and a `main.ts` entry point that wires the form submit, list
render, delete control, and inline error surfaces. Closes every UI-level
acceptance criterion.

## Scope

- `src/web/types.ts` — shared `Bookmark`, `NewBookmarkInput`, `ApiError`.
- `src/web/api.ts`:
  - `fetchBookmarks(): Promise<Bookmark[]>`.
  - `createBookmark(input): Promise<Bookmark>` — POST, parse 201; on
    non-2xx throw `ApiError` carrying `{ code, message }` parsed from
    the body.
  - `deleteBookmark(id): Promise<void>` — DELETE; on non-2xx throw
    `ApiError`.
- `src/web/dom.ts`:
  - `renderBookmarkListItem(b: Bookmark): HTMLLIElement` — produces an
    `<li>` containing an `<a href={b.url} target="_blank"
    rel="noopener noreferrer">{title}</a>` plus a delete `<button>`.
    Title text is set via `textContent` (no innerHTML).
  - `setEmptyState(visible: boolean)` helper.
  - `setFormError(message: string | null)` / `setListError(...)` helpers.
- `src/web/main.ts`:
  - On `DOMContentLoaded`: fetch list, render, toggle empty state.
  - On form submit: validate locally (mirror server rules to avoid a
    pointless round-trip), call `createBookmark`, prepend on success,
    surface inline error keyed off `error.code` on failure (specifically
    `DUPLICATE_URL` → "URL already saved" next to URL field per US-001
    AC4).
  - Delete click: optimistic UI is **not** used — call
    `deleteBookmark`, on success splice out and re-render; on failure
    leave list unchanged and surface inline list error (US-004 AC2).
  - Clear inline errors when the user next edits the relevant input.

## Acceptance criteria delivered

- US-001 AC1 (UI) — form save triggers POST and prepends on success.
- US-001 AC2 (UI) — invalid URL → inline error from server response.
- US-001 AC3 (UI) — empty title → inline error from server response.
- US-001 AC4 (UI) — duplicate URL → "URL already saved" inline error.
- US-002 AC1 — initial fetch renders newest-first as returned by API.
- US-002 AC2 — entries show title and URL.
- US-002 AC3 — newly saved bookmark is prepended without page reload.
- US-002 AC4 — empty-state element shown when list is empty.
- US-003 AC1 — clicking a bookmark opens in a new tab (via
  `target="_blank"`).
- US-003 AC2 — anchor markup includes `rel="noopener noreferrer"`.
- US-004 AC1 — delete control removes the row from SQLite and the list.
- US-004 AC2 — delete failure surfaces an inline error and leaves the
  list unchanged.
