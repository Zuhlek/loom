## Goal

Wire esbuild to produce `public/app.js` from `src/web/main.ts`, and
finalise the `npm start` / `npm test` / `npm run build` scripts so the
whole app runs end-to-end with a single command (per spec constraint
"one-command run").

## Scope

- `app/build.mjs`:
  - Uses esbuild's Node API.
  - Entry: `src/web/main.ts`. Output: `public/app.js`. Format `esm`,
    `bundle: true`, `sourcemap: true`, `platform: 'browser'`, target
    `es2022`.
  - Exits non-zero on build failure.
- `app/package.json` scripts:
  - `"build": "node build.mjs"`.
  - `"start": "node build.mjs && tsx src/server/index.ts"` (per ADR-007).
  - `"test": "vitest run"`.
  - `"typecheck": "tsc --noEmit"`.
- `app/tests/smoke.test.ts`:
  - Boots `buildApp(db)` against an in-memory DB.
  - Asserts `GET /` returns 200 with `Content-Type: text/html`.
  - Asserts `GET /styles.css` returns 200.
  - When `public/app.js` exists (post-build), asserts `GET /app.js`
    returns 200 with a JS content-type; otherwise the assertion is
    skipped with a `it.skipIf(...)` guard to keep CI green pre-build.

## Acceptance criteria delivered

- All four stories satisfied end-to-end: running `npm start` from `app/`
  serves the working UI at `http://localhost:3000` per the seed's
  one-command-run constraint.

## Verification

From `app/`:
- `npm run build` succeeds, `public/app.js` exists.
- `npm test` passes (Vitest suite including the smoke spec).
- `npx tsc --noEmit` passes.
- Optional: `npm start` followed by `curl -sf http://localhost:3000/`
