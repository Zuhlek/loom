# T-002 — SQLite persistence layer

Implement `src/server/db.ts` per design.md `### Server-side function
signatures`:

- `openDb(path: string): BookmarksRepo`
- `BookmarksRepo`: `listAll()`, `insert(input)`, `deleteById(id)`, `close()`
- `Bookmark` and `BookmarkInput` types
- `DuplicateUrlError`, `NotFoundError` classes
- Schema bootstrap via `CREATE TABLE IF NOT EXISTS bookmarks (id INTEGER
  PRIMARY KEY AUTOINCREMENT, url TEXT NOT NULL UNIQUE, title TEXT NOT NULL,
  created_at TEXT NOT NULL)`
- `insert` catches `SqliteError` with `code === 'SQLITE_CONSTRAINT_UNIQUE'`
  and throws `DuplicateUrlError`
- `deleteById` checks `changes === 0` after the prepared delete and throws
  `NotFoundError`
- `listAll` issues `SELECT ... ORDER BY created_at DESC, id DESC`
- `created_at` set in JS via `new Date().toISOString()` at insert time

Path supports `:memory:` for tests and absolute / relative paths for prod.

## Test sketch (Vitest, EARS-derived)

- **US-001 / persistence:** insert a row, `listAll()` returns it with
  matching url/title and a parsable `created_at`
- **US-001 / duplicate:** insert same url twice → second throws
  `DuplicateUrlError`; `listAll().length === 1`
- **US-002 / ordering:** insert three rows, assert order matches
  insertion-newest-first; insert two within the same ms (mock Date.now
  or insert tight loop) and assert id DESC tie-break
- **US-004 / delete:** insert row, `deleteById(id)` succeeds; second
  call throws `NotFoundError`; `listAll()` empty
- **Durability:** open file-backed DB in `os.tmpdir()`, insert,
  `close()`, reopen same path, `listAll()` still returns the row
