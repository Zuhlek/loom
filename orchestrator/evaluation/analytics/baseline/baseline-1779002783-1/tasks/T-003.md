# T-003 — Validation helpers

Implement `src/server/validate.ts` per design.md:

```ts
export type Validated<T> = { ok: true; value: T } | { ok: false; error: string };
export function validateBookmarkInput(raw: unknown): Validated<BookmarkInput>;
```

Rules:

- `raw` must be a non-null object with `title` and `url` properties
- `title`: must be a string; after trim, non-empty; max 500 chars
- `url`: must be a string; must parse via `new URL(...)`; protocol
  must be `http:` or `https:` (rejects `javascript:`, `ftp:`, etc.)
- On success, return the trimmed title and `new URL(...).toString()` as
  the canonical URL string
- On failure, return a human-readable `error` message suitable for
  rendering verbatim in the UI's inline error slot

Pure function only — no I/O, no Express coupling.

## Test sketch (Vitest, EARS-derived)

- **US-001 AC3 (invalid URL):** assert failure for `""`,
  `"not-a-url"`, `"javascript:alert(1)"`, `"ftp://example.com"`, `null`
- **US-001 AC4 (empty title):** assert failure for `""`, `"   "`,
  `undefined`, non-string title
- **Positive paths:** assert success for `"https://example.com"`,
  `"http://example.com/path?q=1#frag"`; assert title trimming and
  canonical URL output
- **Boundary:** title length 500 succeeds; 501 fails
- **Shape:** non-object input (`null`, array, primitive) returns
  `{ ok: false, error }`
