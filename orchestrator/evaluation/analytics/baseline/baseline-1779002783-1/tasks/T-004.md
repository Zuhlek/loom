# T-004 — Express skeleton + static + error mapper

Implement `buildApp(repo: BookmarksRepo): Express` in
`src/server/app.ts`:

- `express.json()` body parser
- Mounts `/api/bookmarks` router from `routes.ts` (handlers are stubs
  in this task — only the router skeleton exists; T-005/T-006/T-007
  fill the methods)
- `express.static('public')` mounted at `/` so `GET /` serves
  `public/index.html`
- Centralised error-handler middleware that maps:
  - `validate.ok === false` (thrown as `ValidationError`) → 400
    `{ error: <message> }`
  - `DuplicateUrlError` → 409
    `{ error: "duplicate", message: "this URL is already saved" }`
  - `NotFoundError` → 404 `{ error: "not found" }`
  - anything else → 500 `{ error: "internal error" }` + `console.error`
- 404 fallthrough for unknown paths (after static + router) returns
  `{ error: "not found" }`

`routes.ts` exports `bookmarksRouter(repo)`; in this task the handlers
exist but return `501 Not Implemented` placeholders (or are absent and
later tasks add them). Either is fine — the goal is a wireable skeleton
that T-005/6/7 plug into without restructuring.
