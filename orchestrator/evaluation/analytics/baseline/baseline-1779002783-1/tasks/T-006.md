# T-006 — POST /api/bookmarks

Implement the create handler in `routes.ts`:

- Parse `req.body`
- Pass through `validateBookmarkInput`; on `ok:false`, throw a
  `ValidationError(error)` (or call `next(new ValidationError(...))`)
  so the centralised mapper returns 400 `{ error }`
- On `ok:true`, call `repo.insert({ url, title })`
- `DuplicateUrlError` bubbles to the mapper → 409 `{ error: "duplicate",
  message: "this URL is already saved" }`
- On success: `res.status(201).json(bookmark)`

## Test sketch (Vitest + supertest, EARS-derived)

- **US-001 AC1 (valid → 2xx + persisted):** POST `{ url, title }` →
  201; response body has `id`, `url`, `title`, ISO `created_at`;
  subsequent `GET /api/bookmarks` includes the new row
- **US-001 AC2 (duplicate → 4xx + no new row):** POST same url twice
  → second is 409 `{ error: "duplicate", message: ... }`;
  `repo.listAll().length === 1`
- **US-001 AC3 (invalid/empty url → 4xx + no row):** POST with
  `url:""`, `url:"not-a-url"`, `url:"javascript:alert(1)"` → each is
  400 `{ error: <string> }`; table remains unchanged
- **US-001 AC4 (empty title → 4xx + no row):** POST with `title:""`,
  `title:"   "`, missing title → 400; table unchanged
- Malformed JSON body → 400 (`express.json` rejects)
