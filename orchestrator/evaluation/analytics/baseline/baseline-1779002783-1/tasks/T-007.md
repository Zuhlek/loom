# T-007 — DELETE /api/bookmarks/:id

Implement the delete handler in `routes.ts`:

- Parse `:id` as a positive integer; non-integer or non-positive →
  fall through to 404 handler (`next()`) so the response shape stays
  `{ error: "not found" }`
- Call `repo.deleteById(id)`
- `NotFoundError` bubbles to mapper → 404 `{ error: "not found" }`
- On success: `res.status(204).end()` (no body)

## Test sketch (Vitest + supertest, EARS-derived)

- **US-004 AC1+AC2 (delete removes row + 2xx):** insert via repo,
  `DELETE /api/bookmarks/:id` → 204 no body; `repo.listAll()` no
  longer contains it
- **US-004 AC3 (missing id → 4xx + unchanged):** `DELETE /api/bookmarks/999`
  on empty DB → 404 `{ error: "not found" }`; `listAll()` unchanged.
  After successful delete, `DELETE /api/bookmarks/<same-id>` → 404 (idempotent failure)
- Non-integer id (`/api/bookmarks/abc`, `/api/bookmarks/-1`,
  `/api/bookmarks/1.5`) → 404
