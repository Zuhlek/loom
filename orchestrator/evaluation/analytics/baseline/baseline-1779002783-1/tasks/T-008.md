# T-008 — Frontend bundle wiring

Stand up the frontend artefact pipeline so later UI tasks (T-009/10/11)
just edit `src/web/*.ts`.

- `app/public/index.html`: minimal HTML5 doc; `<title>Bookmarks</title>`;
  CSS link to `/styles.css`; root container (`<main id="root">`),
  inline-error slot (`<p class="error" id="error" hidden></p>`), form
  (`<form id="new-bookmark">` with `url` + `title` inputs + submit
  button); `<script src="/main.js" defer></script>`
- `app/public/styles.css`: small, no framework; readable list + form;
  inherits `prefers-color-scheme` if it falls out for free
- `src/web/main.ts`: empty `boot()` IIFE that calls `addEventListener('DOMContentLoaded', ...)` — fleshed out in T-009/10/11
- `src/web/api.ts`: typed fetch wrappers (stubs returning `Promise.reject` placeholders are fine; T-009/10/11 fill them)
- `src/web/render.ts`: empty exports per design (stubs)
- `package.json` `build` / `prestart` script wired:
  `esbuild src/web/main.ts --bundle --outfile=public/main.js --format=iife --target=es2020`

This task is foundational for the UI; no user story is directly
delivered by the empty stubs.

## Test sketch

- `npm run build` produces `public/main.js` without errors; file
  exists and is non-empty
- `npm start` (when later wired) serves `GET /` → 200 HTML; `GET
  /main.js` → 200 JS; `GET /styles.css` → 200 CSS (asserted in T-005
  or T-012 via supertest against the static middleware)
- `tsc --noEmit` passes against the stubs
