# T-009 — Frontend list rendering + empty state + open-in-new-tab

Wire the read path:

- `api.ts`: implement `fetchBookmarks(): Promise<Bookmark[]>` calling
  `GET /api/bookmarks`; throw `{ kind: 'network' }` on non-200
- `render.ts`:
  - `renderList(root, items)`: replaces `root` children with one row
    per item; each row contains an `<a href={item.url} target="_blank"
    rel="noopener noreferrer">{item.title}</a>` (title via
    `textContent` to avoid XSS); plus a `<button data-id={item.id}
    data-action="delete">Delete</button>` placeholder (handler in T-011)
  - `renderEmptyState(root)`: writes an empty-state message like "No
    bookmarks yet. Add your first one above."
- `main.ts`: on `DOMContentLoaded`, call `fetchBookmarks()`, then
  branch: empty → `renderEmptyState`, otherwise → `renderList`

## Test sketch (EARS-derived)

- **US-002 AC1+AC2:** end-to-end already covered by T-005's api.test.ts
  (server returns rows in newest-first order); render layer trusted by
  visual review + bundle inspection (`public/main.js` contains the
  literal `target="_blank"` and `rel="noopener noreferrer"`)
- **US-002 AC3 (empty-state):** unit test for `renderEmptyState` —
  inject a detached `<main>` element, call the function, assert the
  rendered text matches the empty-state copy
- **US-003 AC2 (anchor attributes):** unit test for `renderList` over
  one item — assert the rendered anchor has `target === "_blank"` and
  `rel === "noopener noreferrer"`; title is set via `textContent`
  (so HTML injection is impossible)
- Bundle smoke: after `npm run build`, the output contains
  `target="_blank"` and `rel="noopener noreferrer"`
