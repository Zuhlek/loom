# T-010 — Frontend create form + inline error handling

Wire the create path:

- `api.ts`: implement `createBookmark(input): Promise<Bookmark>` calling
  `POST /api/bookmarks`. Behaviour per design.md:
  - 201 → resolve with parsed body
  - 409 → throw `{ kind: 'duplicate' }`
  - 400 → throw `{ kind: 'validation', message: <body.error> }`
  - other → throw `{ kind: 'network' }`
- `render.ts`: implement `renderInlineError(slot, message)` (sets text,
  removes `hidden`) and `clearInlineError(slot)` (clears text, sets
  `hidden`)
- `main.ts`: attach `submit` handler on `#new-bookmark` form
  - `preventDefault()`
  - read url + title inputs
  - call `createBookmark`
  - on success: reset form, `clearInlineError`, re-fetch + re-render list
  - on `duplicate`: `renderInlineError(slot, "this URL is already saved")`
  - on `validation`: `renderInlineError(slot, error.message)`
  - on `network`: `renderInlineError(slot, "network error")`
  - on any form `input` event: `clearInlineError`
