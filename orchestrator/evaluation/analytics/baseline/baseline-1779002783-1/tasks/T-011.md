# T-011 — Frontend delete control + re-fetch

Wire the delete path:

- `api.ts`: implement `deleteBookmark(id: number): Promise<void>` calling
  `DELETE /api/bookmarks/:id`; resolve on 204; throw `{ kind: 'network' }`
  on non-204 (404 here means the row already disappeared — treat as
  benign and re-fetch anyway)
- `main.ts`: attach a delegated `click` handler on the list root that
  matches `[data-action="delete"]`, reads `data-id`, calls
  `deleteBookmark(id)`, then re-fetches + re-renders the list. On
  network failure, render inline error "network error".

The delete button itself is rendered by `renderList` in T-009; this
task only adds the handler.
