# T-012 — npm start entrypoint + workspace-isolation smoke

Land the last piece of glue and the acceptance-gate guard.

- `src/server/index.ts`: opens the file-backed DB at `./bookmarks.db`,
  calls `buildApp(repo)`, listens on `:3000`. Logs `"listening on
  http://localhost:3000"` on bind. Exits non-zero on any throw from
  `openDb` or `listen`.
- `package.json` `start` script: runs the esbuild step (idempotent if
  `public/main.js` is already up to date is fine — simplest is always
  build) then `node --import tsx src/server/index.ts`.
- `tests/smoke.test.ts` (Vitest):
  - Spawn the server in a child process from `./app/` with a tmpdir
    `cwd` so it writes to a throwaway DB
  - Wait for the "listening" line on stdout (with a generous timeout)
  - HTTP GET `http://localhost:3000/api/bookmarks` → 200 `[]`
  - HTTP GET `http://localhost:3000/` → 200, HTML body
  - POST a bookmark, GET, assert it appears
  - DELETE it, GET, assert empty
  - Kill child, assert clean exit
- **Workspace isolation guard:** the same test or a parallel one
  asserts that no files were written outside the tmpdir / `./app/`.
  Concretely: snapshot `git status --porcelain` paths (or stat the
  repo root and `orchestrator/` for mtimes) before and after; any
  drift fails the test with a "wrote outside ./app/" message.

This task closes US-001..US-004 end-to-end as a single smoke gate
running against the real file-backed process.

## Test sketch (EARS-derived)

- All four user stories exercised in one process: create, list,
  open-attribute presence in rendered HTML/bundle, delete
- Bundle inspection: rendered list HTML (after a POST + GET) — the
  bundle's serialised behaviour produces anchors with `target="_blank"`
  and `rel="noopener noreferrer"`
- Isolation: no writes outside `./app/` and the tmpdir; `git status`
  on `orchestrator/` and repo root unchanged from the pre-test snapshot
- `npm start` exits non-zero if `:3000` is already bound (sanity)
