# T-001: Scaffold `app/` workspace

## Objective

Create the `.loom/baseline-1779002783-2/app/` directory and every file
required for `npm install`, `npm start`, and `npm test` to be valid
commands (even before they run successfully). This task establishes
the workspace substrate every subsequent task depends on.

## Deliverables

- `app/package.json` declaring:
  - `"name": "baseline-1779002783-2"`, `"type": "module"`.
  - Engines: `"node": ">=20"`.
  - Dependencies: `express` (^4), `better-sqlite3` (^11).
  - DevDependencies: `typescript` (^5), `tsx` (^4), `esbuild`
    (^0.23+), `vitest` (^2), `supertest` (^7), `@types/express`,
    `@types/supertest`, `@types/node`, `@types/better-sqlite3`,
    `happy-dom`.
  - Scripts: `start`, `build:client`, `build:server`, `build`,
    `test`, `smoke`. `start` runs `build:client` then launches the
    server via `tsx src/server/index.ts`.
- `app/tsconfig.json` for the server compile: `strict: true`, `target:
  ES2022`, `module: NodeNext`, `moduleResolution: NodeNext`, `outDir:
  dist`, `rootDir: src`, `esModuleInterop: true`.
- `app/tsconfig.client.json` (extends the base) with `module: ESNext`,
  `moduleResolution: Bundler`, `noEmit: true` — esbuild owns client
  emit, but type-checking should still run.
- `app/scripts/build-client.mjs`: imports `esbuild`, bundles
  `src/client/main.ts` → `public/bundle.js`, format `esm`, target
  `es2022`, sourcemap inline. Idempotent; skips rebuild if output is
  newer than inputs (ADR-008). Exits non-zero on failure.
- `app/.gitignore`: `node_modules/`, `dist/`, `public/bundle.js`,
  `bookmarks.sqlite*`.
- `app/vitest.config.ts`: sets `environment: 'happy-dom'` for client
  tests; `include` covers `tests/**/*.test.ts`.

## Acceptance gates

- `cd app && npm install` exits 0.
- `cd app && npm test` exits 0 (with a placeholder `tests/_init.test.ts`
  asserting `expect(true).toBe(true)`).
- `cd app && node scripts/build-client.mjs` exits 0 when
  `src/client/main.ts` is a stub `console.log("ok")`. The output
  `public/bundle.js` exists afterwards.
- No file from this task lives outside `app/`.

## Single-layer justification

This task is single-layer (build / config) because no observable user
behaviour can land without the workspace existing. Splitting it
vertically into per-story scaffolding would duplicate `package.json`
and confuse Build. It is tagged `US-005` because the `npm start` entry
point and the SQLite file-path resolution it wires (`./bookmarks.sqlite`
relative to `process.cwd()`, which is `./app/`) are the structural
preconditions for survive-restart persistence; no other story can
deliver value without this scaffolding.

## Test sketch

- Vitest placeholder asserts the runner is wired correctly.
- Build script invocation is asserted by Build's pre-flight rather than
  by an in-suite test (running esbuild from inside Vitest would couple
  test runtime to build runtime unnecessarily).

## Notes

- `tsx` is used for `npm start` to avoid an ahead-of-time server
  compile during the seed run; the seed never claims a compiled-JS
  start path. If Review requires `tsc` precompile, swap `tsx` for
  `node dist/server/index.js` and add `build:server` to `start`.
- `better-sqlite3` is a native module; `npm install` will compile it
  via `node-gyp`. Build's pre-flight must allow this.
