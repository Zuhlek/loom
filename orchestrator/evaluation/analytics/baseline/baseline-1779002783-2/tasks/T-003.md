# T-003: List bookmarks slice

## Objective

Vertical slice that makes `GET /api/bookmarks` return every persisted
bookmark in `created_at DESC, id DESC` order. First slice that creates
the `createApp` factory and the bookmarks router; T-004 and T-005
extend that scaffolding.

## Deliverables

- `src/server/repo/bookmarks.ts`:
  - `BookmarkRow` type per `design.md ## Server-side function
    signatures`.
  - `listBookmarks(db)` prepared statement: `SELECT id, url, title,
    created_at FROM bookmarks ORDER BY created_at DESC, id DESC`.
  - Row → response shape mapping (`created_at` → `createdAt`).
- `src/server/routes/bookmarks.ts` exposing `bookmarksRouter(db)`
  with `GET /` returning `{ bookmarks: [...] }`. Other verbs come in
  T-004/T-005.
- `src/server/app.ts` exposing `createApp(db)`:
  - Mounts `bookmarksRouter(db)` at `/api/bookmarks`.
  - Wires `express.json({ limit: '32kb' })` and `express.static('public')`
    (T-006 lands `public/`).
  - Single error middleware mapping unknown `Error` → `500 { error: {
    code: "internal" } }` and logging stack to stderr.

## Acceptance criteria covered

- **US-002 AC-1** — `GET /api/bookmarks` returns every row in
  `created_at DESC, id DESC` order.

## Test sketch

### Unit (`tests/unit/repo.test.ts` — partial)

- `listBookmarks(db)` on empty DB returns `[]`.
- After three inserts at distinct timestamps, ordering is descending
  by `created_at`, with `id` as tiebreaker for identical timestamps.

### Integration (`tests/integration/bookmarks.api.test.ts` — partial)

- `GET /api/bookmarks` on a fresh `:memory:` DB → `200 { bookmarks:
  [] }`.
- After seeding three rows via the repository, `GET` returns them in
  newest-first order.
- Response Content-Type is `application/json`.

## Notes

- Empty-state UI copy is owned by T-006 (client). This slice only
  guarantees the server reports `bookmarks: []` cleanly.
