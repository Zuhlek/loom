# T-004: Create bookmark slice

## Objective

Vertical slice that makes `POST /api/bookmarks` validate input, insert
a row, return `201 { bookmark }`, reject duplicates with `409`, and
reject invalid input with `400`. Establishes the validation module
that T-005 also consumes for `:id` parsing.

## Deliverables

- `src/server/validate.ts`:
  - `validateUrl(raw): { ok: true; value: string } | { ok: false; reason: string }`.
    Rules per `design.md ## Validation rules`: non-empty after trim,
    length ≤ 2048, starts with `http://` or `https://`, parses via
    `new URL(value)`.
  - `validateTitle(raw): { ok: true; value: string } | { ok: false; reason: string }`.
    Non-empty after trim, length ≤ 512.
  - `parseId(raw): number | null`. Rejects non-`/^[1-9]\d*$/`.
- `src/server/repo/bookmarks.ts` extended with:
  - `DuplicateUrlError extends Error { url: string }`.
  - `createBookmark(db, { url, title })`: `INSERT` via prepared
    statement; on `SQLITE_CONSTRAINT_UNIQUE` throws
    `DuplicateUrlError`. Returns the freshly inserted `BookmarkRow`
    (using `lastInsertRowid` and a follow-up `SELECT`).
- `src/server/routes/bookmarks.ts` extended with `POST /`:
  - Rejects non-JSON `Content-Type` → `415`.
  - Validates `url`, `title`; on failure → `400 { error: { code:
    "invalid_input", field, message } }`.
  - Catches `DuplicateUrlError` → `409 { error: { code:
    "duplicate_url", url, message } }`.
  - Success → `201 { bookmark: {...} }`.

## Acceptance criteria covered

- **US-001 AC-1** — persists bookmark; visible in next list response.
- **US-001 AC-2** — duplicate URL → `409` + structured error body
  (the inline-error UI is owned by T-007).
- **US-001 AC-3** — empty / non-`http(s)` URL → `400`.
- **US-001 AC-4** — empty / whitespace-only title → `400`.

## Test sketch

### Unit (`tests/unit/validate.test.ts`)

- `validateUrl`: empty, whitespace-only, missing scheme,
  `ftp://example`, `javascript:alert(1)`, oversize (length 2049),
  malformed (`https://`), valid `http://example.com`, valid
  `https://example.com/path?q=1#frag`.
- `validateTitle`: empty, whitespace-only, oversize, single-character.
- `parseId`: `'0'`, `'-1'`, `'1.5'`, `'1e2'`, `'01'`, `''`, valid
  `'42'`.

### Unit (`tests/unit/repo.test.ts` — extended)

- `createBookmark` returns row with `id` and ISO-8601 `createdAt`.
- Second `createBookmark` with identical URL throws `DuplicateUrlError`
  whose `.url` matches.
- Title and URL are stored exactly as submitted post-trim.

### Integration (`tests/integration/bookmarks.api.test.ts` — extended)

- `POST` happy path → `201`; `GET` shows the row first.
- `POST` duplicate → `409 { error: { code: "duplicate_url", url } }`.
- `POST` empty URL → `400 { error: { code: "invalid_input", field:
  "url" } }`.
- `POST` `url: "ftp://x"` → `400`.
- `POST` `title: "   "` → `400 { ... field: "title" }`.
- `POST` with `Content-Type: text/plain` → `415`.
