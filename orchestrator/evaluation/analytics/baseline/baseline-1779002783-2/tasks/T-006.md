# T-006: Static shell + client bootstrap

## Objective

Establish the static HTML/CSS shell, the client bundle entry, the
`api` and `render` modules, and the server `index.ts` bootstrap that
wires `db.ts` into `app.ts`, runs `build-client.mjs` if `bundle.js` is
missing or stale, and calls `app.listen(3000)`. Renders empty / loading
/ populated states for the list.

## Deliverables

- `public/index.html`: minimal shell with `<form id="new">`
  (url + title + submit), `<aside id="error" hidden></aside>`,
  `<ul id="bookmarks"></ul>`, `<script type="module" src="/bundle.js">`.
- `public/styles.css`: hand-written, no preprocessor. Hides `#error`
  via `[hidden]`; sane defaults; readable on default user-agent fonts.
- `src/client/api.ts`:
  - `fetchBookmarks(): Promise<Bookmark[]>` — `GET /api/bookmarks`.
  - `createBookmark(url, title)` — `POST` with JSON body. Throws
    `ApiError` on non-2xx with `{ status, code, field? }`.
  - `deleteBookmark(id)` — `DELETE`. Throws `ApiError` on non-2xx.
  - `ApiError extends Error { status: number; code: string; field?:
    string }`.
- `src/client/render.ts`:
  - `renderList(container, bookmarks)`: if empty, render
    `<li class="empty">No bookmarks yet — add one above.</li>`; else
    one `<li>` per bookmark with an `<a target="_blank" rel="noopener
    noreferrer">` for the title and a separate URL text node and a
    delete `<button data-id="...">`. All text via `textContent`.
  - `renderError(container, message)`: sets `textContent`, removes
    `hidden`.
  - `clearError(container)`: clears text, sets `hidden`.
- `src/client/main.ts`:
  - On `DOMContentLoaded`: renders "Loading…", calls
    `fetchBookmarks`, calls `renderList`. T-007/T-008/T-009 add the
    form/delete wiring.
- `src/server/index.ts`:
  - Imports `openDatabase`, `migrate`, `createApp` from `src/server/`.
  - On boot: runs `build-client.mjs` (via `node:child_process spawnSync`
    or by importing the script) if `public/bundle.js` is missing or
    older than any input.
  - `migrate(db)` then `createApp(db).listen(3000)`.
  - Wires `SIGINT`/`SIGTERM` to close server and `db.close()`.

## Acceptance criteria covered

- **US-002 AC-1** — list renders in newest-first order (relies on
  server ordering from T-003).
- **US-002 AC-2** — empty-state copy renders instead of an empty list.
- **US-002 AC-3** — each row displays title + URL.

## Test sketch

### Unit (`tests/unit/client/render.test.ts`)

- Empty input → `<li class="empty">` text matches the seed copy.
- Three bookmarks → three `<li>` elements; titles and URLs appear via
  `textContent` (no HTML injection — assert with a
  `<script>alert(1)</script>` title that the rendered DOM has zero
  `<script>` children).
- Anchor has `target="_blank"` and `rel="noopener noreferrer"`.
- `renderError` + `clearError` toggle the `hidden` attribute.

### Unit (`tests/unit/client/api.test.ts`)

- `fetchBookmarks` returns the parsed `bookmarks` array.
- `createBookmark` posts the right body and Content-Type.
- 409 response → `ApiError` with `code === "duplicate_url"`.
- 400 with field → `ApiError` with `field` populated.

## Single-layer justification

T-006 spans client + static + a thin server bootstrap, so it is
*not* single-layer. It is grouped because the static shell, the
client entry, and the server's `build-client + listen` glue must land
together for the page to load at all. Splitting them would create
three tasks that each fail Build's smoke until all three are merged.
