# T-008: Client list render + open-in-new-tab

## Objective

Solidify the populated-state rendering: ordering as received from the
server (server is authoritative on ordering), title as anchor with
`target="_blank"` + `rel="noopener noreferrer"`, URL displayed
alongside, delete button stub for T-009. Text-content-only insertion
(no `innerHTML`).

## Deliverables

- `src/client/render.ts` finalised:
  - One `<li>` per bookmark, in array order.
  - Title is wrapped in `<a href="${url}" target="_blank"
    rel="noopener noreferrer">` — `href` set via `setAttribute` /
    property; title text via `textContent`.
  - URL rendered in a separate inline `<span class="url">` via
    `textContent`.
  - Delete `<button data-id="${id}">Delete</button>` placeholder; T-009
    attaches its handler.

## Acceptance criteria covered

- **US-002 AC-1** — list ordering preserved on render (server returns
  newest-first; client does not re-sort).
- **US-002 AC-3** — title and URL both displayed.
- **US-003 AC-1** — anchor opens in a new tab.
- **US-003 AC-2** — every anchor carries `rel="noopener noreferrer"`.

## Test sketch (`tests/unit/client/render.test.ts` — extended from T-006)

- Render three bookmarks; assert array order preserved on the
  `<li>` children.
- Each anchor has the expected `href`, `target`, and `rel`.
- Insertion is via `textContent` — assert that a bookmark with a
  `title` of `<img onerror=alert(1)>` produces no `<img>` element.
- `<span class="url">` text equals the input URL exactly.
- Delete button carries `data-id` matching the bookmark id.
