# T-009: Client delete control + inline not-found

## Objective

Wire delete buttons (added by T-008) to `api.deleteBookmark`,
re-fetch on success, and surface a 404 error inline per US-004 AC-2.

## Deliverables

- `src/client/main.ts` extended:
  - Event delegation on `#bookmarks` for `click` events whose target
    matches `button[data-id]`.
  - Reads `data-id`; calls `api.deleteBookmark(id)`.
  - On success: clears `#error`, calls `fetchBookmarks` + `renderList`.
  - On `ApiError` with `code === "not_found"`: renders `"That
    bookmark is no longer there — refreshing."` into `#error`, then
    calls `fetchBookmarks` to re-sync.
  - Other `ApiError`: generic `"Couldn't delete: ${message}"`.

## Acceptance criteria covered

- **US-004 AC-1** — successful delete removes row on next render.
- **US-004 AC-2** — 404 surfaces inline.

## Test sketch (`tests/unit/client/delete.test.ts`)

- Under `happy-dom`: render two bookmarks; click delete on first;
  mock `fetch` to return `204`. Assert `fetchBookmarks` is called and
  `#error` stays hidden.
- Same setup, mock `fetch` to return `404 { error: { code:
  "not_found", id } }`. Assert `#error` is visible with the recovery
  message, and a re-sync `fetch` is issued.
- Network failure → `#error` shows generic message.
