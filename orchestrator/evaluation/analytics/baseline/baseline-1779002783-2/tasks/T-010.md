# T-010: End-to-end smoke

## Objective

Prove that the full stack boots from `npm start`, serves the JSON
API, and survives a restart with state intact — the only autonomous
gate that covers US-005 AC-1 (persistence across server restarts).
Operates entirely under the `node-test` capability; no browser.

## Deliverables

- `scripts/smoke.mjs`:
  - Creates a unique temp dir; sets `BOOKMARKS_DB` to a file in it.
  - Spawns `npm start` (or `node --import tsx src/server/index.ts`
    directly) with that env. Captures stdout/stderr for diagnostics.
  - Polls `GET http://localhost:3000/api/bookmarks` for up to 10 s
    until `200`.
  - POSTs `{ url: "https://example.com/smoke", title: "smoke" }`;
    asserts `201`.
  - GETs; asserts the row appears.
  - Sends `SIGINT`; awaits exit.
  - Respawns the server against the same `BOOKMARKS_DB`.
  - Polls; GETs; asserts the smoke row is still present.
  - DELETEs the row; asserts `204`; GETs; asserts empty array.
  - Cleans up the temp dir; exits 0 on success, 1 on any assertion
    failure.
- `package.json` script: `"smoke": "node scripts/smoke.mjs"`.
- `tests/integration/smoke.test.ts` (optional): a Vitest wrapper that
  shells out to `npm run smoke` and asserts exit 0. Build can run
  this as part of `npm test` if it sequences correctly with port
  3000; otherwise Build invokes `npm run smoke` separately.

## Acceptance criteria covered

- **US-005 AC-1** — restart cycle proves persistence.
- **US-005 AC-2** — first-boot cold start with no SQLite file proves
  the migration created it (the smoke uses a fresh temp dir, so the
  file does not exist on first spawn).

## Test sketch

The script *is* the test. It asserts:

- First spawn produces a SQLite file at `$BOOKMARKS_DB`.
- POST then GET returns the smoke row.
- Second spawn (after SIGINT) returns the same row from disk.
- DELETE cycle empties the list.

## Notes

- Port 3000 is hardcoded by the seed. The smoke must not run in
  parallel with the Vitest suite if Vitest also boots the server (it
  does not — Vitest uses `:memory:` and `supertest`, never binds
  port 3000). Build can run them sequentially: `npm test && npm run
  smoke`.
- Failure mode to watch: `better-sqlite3` native build on the Build
  coordinator. If `npm install` fails to compile it, smoke cannot
  run; Build returns `blocked` with the build-tool gap.
